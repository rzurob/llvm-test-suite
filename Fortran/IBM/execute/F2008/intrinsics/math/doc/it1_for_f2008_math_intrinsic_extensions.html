Overview

%Note{Simple description of the feature; In which product it is first implemented, is it common for other platforms, is it common for both C and C++ and so on.}%

This document describes the testing of the following extensions and new math intrinsics introduced in Fortran 2008 standard:

Extensions:

  • Complex numbers support added in ACOS, ASIN, ATAN, COSH, SINH, TAN and TANH.
  • GAMMA is no longer IBM extension.
  • ATAN2 can be referenced by the name ATAN.

New intrinsics:

  • ACOSH, ASINH, ATANH and LOG_GAMMA.

For details of these intrinsics reader is referred to the Fortran standard [1].


Test Strategy

%Note{Test strategy used in the test}%

Following are some of the considerations while writing the new testcases (TCs):

Diagnostic:

  • Test the new intrinsics with different arguments including real, complex, integer, character, string, logical etc.
  • Test intrinsics as mentioned in I0's Section "User Documentation --> Messages" with different language levels including 77, 90, 95, 2003 etc. Warning message to be checked:
    • 1518-256 (L) The intrinsic procedure <name> (with these arguments) is not permitted by the Fortran <level> standard.
  • Test the new LOG_GAMMA intrinsic procedure with negative and zero values.
  • Test ATAN intrinsic procedure with two arguments (as in ATAN2).

Functional:

  • Test the result values of the following intrinsic procedures if they are within the range specified in the Fortran standard [1] by comparing the exact precise values using precision functions:
    • ACOS, ASIN, ATAN, COSH, SINH, TAN and TANH with only complex arguments.
    • ACOSH, ASINH and ATANH with both real (single and double precision) and complex arguments. LOG_GAMMA with only real (single and double precision) arguments.
  • Test the use of intrinsic procedures with both scalar and array arguments.
  • Test the use of intrinsic procedures in an initialization expression (e.g: an expression inside an ASSOCIATE construct), a module, a function, a subroutine and an elemental function with array arguments.
  • This is how the results will be verified:
    • Calling the intrinsic in an initialization expression and an expression which is not an initialization expression with the same argument values. The results will be compared bit by bit and should be same.
    • Calling the intrinsic and the reciprocal of the intrinsic using the result returned from the intrinsic.
            e.g:
            RES = acos(1.0, 0.0)
            cos(RES)                    ! should be equal to 1.0
    • Comparing against an already known value.
