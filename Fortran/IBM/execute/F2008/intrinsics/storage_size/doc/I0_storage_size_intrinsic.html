Feature Overview

Fortran 2008 standard introduced a new intrinsic STORAGE_SIZE that returns the size expressed in bits for an element of an array that has the dynamic type and type parameters of its argument.

Key Requirements

To implement this new intrinsic.

Key Design Issues

Since the intrinsic returns the size of the object as the element of an array as if it were to store into the memory, determine the padding and interaction with -qautodbl, -qinitsize or -qrealsize is important.

Key Design Decisions

The compiler already takes a variety of options into account when setting up the length at both compile time and runtime. It sets the compile time length or the length field in the descriptor for runtime correctly. This intrinsic then can be folded at the compile time.

Testing Strategy

This is a unit test only feature.

For diagnose testing, invalid argument (expression, array constructor, structure constructor) should trigger a compile time message.
For diagnose testing, a langlvl message is issued when the -qlanglvl option specifies a standard prior to F2008.
Fort function testing,
  • scalar or array of different intrinsic types and derived types.
  • variables with allocatable or pointer attribute.
  • polymorphic entities.
  • assumed-length dummy argument.
  • runtime length automatic object.
  • interaction with -qintsize, -qrealsize
  • interaction with -qautodbl option

 

User Documentation

Overview

Fortran 2008 standard introduced a new intrinsic STORAGE_SIZE that returns the size expressed in bits for an element of an array that has the dynamic type and type parameters of its argument.

User Benefits

It allows users to retrieve the size of a data object as if it were to store to the memory.

Public Interface

STORAGE SIZE (A [, KIND])

Description. Storage size in bits.

Class. Inquiry function.

Arguments.
A shall be a data object of any type. If it is polymorphic it shall not be an undefined pointer. If it is unlimited polymorphic or has any deferred type parameters, it shall not be an unallocated allocatable variable or a disassociated or undefined pointer.
KIND (optional) shall be a scalar integer constant expression.

Result Characteristics. Integer scalar. If KIND is present, the kind type parameter is that specified by the value of KIND; otherwise, the kind type parameter is that of default integer type.

Result Value. The result value is the size expressed in bits for an element of an array that has the dynamic type and type parameters of A. If the type and type parameters are such that storage association (16.5.3) applies, the result is consistent with the named constants defined in the intrinsic module ISO FORTRAN ENV.

Constraints

The argument of STORAGE_SIZE must be a date object, namely, constant, variables or subobject of constant. It cannot be an expression, an array constructor, a structure constructor.

High-Level Implementation Design

Implementation Design Overview

Fortran FE will always fold the intrinsic. It should always return the element length of the entity. If it is known at the compile time, FFE will get it from SS(arg_stk).lngth. If it is only known at the runtime, FFE will load the argument and call __len_with_kind. ASTI will then get the value from the element_len field of the runtime descriptor.
