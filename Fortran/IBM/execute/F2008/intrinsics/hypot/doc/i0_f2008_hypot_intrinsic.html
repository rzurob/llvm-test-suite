Feature Overview

The Fortran 2008 HYPOT intrinsic calculates the Euclidean distance between two values.

Key Requirements

The Fortran 2008 standard requires that the calculation of HYPOT not suffer from undue overflow or underflow.

Key Design Issues

  1. Avoiding undue overflow and underflow
  2. libm Compatibility

Key Design Decisions

We will use the POSIX hypotf, hypot, and hypotl functions to compute the result. These functions are provided in libm and are designed to avoid undue overflow and underflow. Using these functions also allows us a program calling the Fortran HYPOT to get the same results as a program (or part of a mixed Fortran-C program) calling the libm hypot.

AIX does not provide a 128-bit long double implementation of hypotl. The AIX developers are aware of the issue. No action will be taken unless, we get customer demand for such a function. Similar to the Fortran 2008 complex trig functions, we will give a "not supported" message for the real(16) version of the funtion on AIX.  Robert Enenkel will write a 128-bit long double implementation of hypotl, and we'll contribute it to AIX.  Since AIX 5.3 is not getting any new TLs, his implementation can only be contributed to AIX 6.1 and AIX 7.1.  So we'll try to get his implementation into AIX 6.1 and AIX 7.1 as soon as possible, and once we drop support for AIX 5.3 in AIX 15.1, we'll require the AIX 6.1 and AIX 7.1 TLs containing his function.

Other approaches that I considered were:

  • Don't flag hypot(real(16)) as unsupported. The user will get an undefined symbol error form the linker, similar to what they currently get from the C compiler. The advantage of this approach is that no action on the part of the compiler is needed once AIX integrates Robert's function. The disadvantage is that having a linker error for this is not user-friendly and will likely cause PMRs, and at least a Tech Note.
  • Add Robert's function to libxl.a under a different name. The reason I dislike this approach is that we'd have to ensure that the libm version, once it's available returns exactly the same results as the libxl.a version. We'd also have to modify the compiler to call the libm version.

User Documentation

Overview

The Fortran 2008 HYPOT intrinsic calculates the Euclidean distance between two values.

User Benefits

The Fortran 2008 HYPOT intrinsic provides a standard, portable way of calculating Euclidean distance while avoiding undue overflow and underflow.

Public Interface

HYPOT(X, Y) (Fortran 2008)

Purpose
Euclidean distance between two values

Class
Elemental function

Argument type and attributes

X
must be of type real
Y
must be of the same type and kind type parameter as X

Result type and attributes
Same as X

Result value
The result value is equal to sqrt(x**2 + y**2), approximately.

Example
HYPOT(3.0, 4.0) has the value 5.0

Constraints
  • The actual arguments corresponding to X and Y must be of type real and must have the same kind type parameter value.
  • On AIX, X and Y must not be of type REAL(16). (Note to ID: Don't document this one)

Rules

The HYPOT intrinsic function is allowed in constant expressions.

Language Standard Considerations

The HYPOT intrinsic conforms to the Fortran 2008 standard. It will be flagged with a language level message when checking for conformance with earlier standards.

Messages

No new messages are needed. Existing messages cover all constraints and language level checking.

Performance Considerations

HYPOT is implemented by calling the libm function, so it has the same performance. MASS provides versions of hypot and hypotf, so making the calls to the libm signatures allows the use of the MASS replacements when linking with the MASS libraries.

Multi-Platform/Portability Considerations

The implementation returns the same result as the libm functions. (So it doesn't matter if the user calls hypot from the C or Fortran portions of their programs or libraries.

HYPOT(X, Y) where X and Y are of type REAL(16) is not supported on AIX.

Migration/Co-Existence/Binary-Compatibility Considerations

None

Debug Considerations

None

NLS (Unicode) Considerations

N/A

Samples

HYPOT(3.0, 4.0) has the value 5.0


Architecture Review Board

No review needed. The interface of this function is described by the standard. The implementation is done in libm, and deciding to choose libm is consistent with what we've been doing with other intrinsics.

Inter-Component Communication/Interfaces

hypot will results in calls to one of the following libm functions for real(4), real(8), and real(16) arguments, respectively:

  • On AIX: hypotf, hypot, (not allowed)
  • On Linux and BGQ: hypotf, hypot, hypotl
