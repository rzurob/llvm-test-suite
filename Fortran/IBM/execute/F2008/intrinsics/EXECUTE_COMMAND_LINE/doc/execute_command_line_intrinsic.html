 Access Change History

Feature Overview 
     Fortran 2008 adds the EXECUTE_COMMAND_LINE intrinsic which is similar to XLF's SYSTEM intrinsic(which is an IBM extension) 
 and C's "system" system call. Users can invoke the intrinsic to execute a command line.

Key Requirements

     EXECUTE_COMMAND_LINE is a required feature to comply with the Fortran 2008 standard.

Key Design Issues

     (1)XLF will only support synchronous execution. 
     (2)BlueGene does not support command line execution.

Key Design Decisions

     (1)We call system() routine to execute a command line, system() calls fork() to produce child process to execute
 a command. System() shall ignore the SIGINT and SIGQUIT signals, and shall block the SIGCHLD signal, it won't return
 until the child process has terminated. Therefore, the command is executed synchronously. Since EXECUTE_COMMAND_LINE
 calls system(), XLF will only support synchronous execution.
     (2)Since BlueGene does not support command line execution and the standard clearly defines the behavior when command
 line execution is not supported, we do not need to do anything special for BlueGene. We will not disallow this intrinsic 
 for BlueGene, but the program will always set CMDSTAT to -1 if it is present, or the program will be terminated if 
 CMDSTAT is not present.

Testing Strategy

 unit testing:
  • Functional: Test that EXECUTE_COMMAND_LINE can be compiled successfully with different combination of arguments. It can execute the command that user specified correctly and set EXITSTAT, CMDSTAT and CMDMSG corrrectly if they are present. Test that the program will be terminated when a condition occurs that would assign a nonzero value to CMDSTAT but the CMDSTAT variable is not present.
  • Diagnostic: We need a diagnostic unit test for testing -qlanglvl=<older standards>, we'll give a langlvl message(L) when the subroutine is used with langlvl 2003 and below, but the subroutine should still work. Also, verify the correctness of the message text of CMDMSG if it is present. 
 xlftest strategy:
  • The feature needs both Diagnostic and Functional testing. The diagnostic test will ensute that the intrinsic subroutine EXECUTE_COMMAND_LINE will behave according to the specification of the F2008 standard. In particular that no error message is emitted when EXITSTAT, CMDSTAT and CMDMSG are omitted or present. Test that we diagnose mismatching type and rank of arguments. Test that if no error condition occurs, then if WAIT is specified and set to .FALSE., set CMDSTAT to -2, otherwise set CMDSTAT to 0.
  • The test strategy for the functional testing is to verify the correct execution of the program or the command line started by the EXECUTE_COMMAND_LINE intrinsic. Different Unix/Linux commands will be used (ie. ECHO, mv, date,...). Different verification files might need to be generated depending on the platform. Running a different Fortran program will also be tested using EXECUTE_COMMAND_LINE.
  • When EXITSTAT, CMDSTAT and CMDMSG are present, we need to ensure that the exit value is correct and conform to the expected behavior specified in the F2008 standard.
  • The feature only involves adding one new intrinsic, it won't have any appreciable impact on performance. Hence high optimization option testing is unnecessary.

User Documentation 
     Fortran 2008 adds the EXECUTE_COMMAND_LINE intrinsic which is similar to XLF's SYSTEM intrinsic(which is an IBM extension) 
 and C's "system" system call. Users can invoke the intrinsic to execute a command line.

User Benefits

     Users now can call EXECUTE_COMMAND_LINE intrinsic to execute a specified command line.

Public Interface

     Class: SUBROUTINE    
 
     EXECUTE_COMMAND_LINE(COMMAND, WAIT, EXITSTAT, CMDSTAT, CMDMSG)
                               
     COMMAND: must be scalar and of type character, specifying the command to execute and any command-line arguments.
 It is an INTENT (IN) argument. 
 
     WAIT (optional): must be scalar and of type logical, specifies whether to allow execution synchronously or asynchronously.
 It is an INTENT (IN) argument. 
     XLF always execute the command synchronously.
 
     EXITSTAT (optional): must be scalar and of type integer, when present it will be set to the exit code returned by the
 executed command. It is an INTENT (INOUT) argument. 
 
     CMDSTAT (optional): must be scalar and of type integer. When present, it will be assigned a value indicating the status
 of the command execution as described below:
     If command line execution is not supported, it is assigned the value -1.
     If an error condition occurs it is assigned the value 1.
     If no error condition occurs but WAIT is present with the value false, it is assigned the value -2.
     In all other cases it is assigned the value 0.  
 It is an INTENT (OUT) argument. 
 
     CMDMSG (optional): must be scalar and of type character. An explanatory message is assigned to this argument when 
 it is present and an error condition occurs. It is an INTENT (INOUT) argument.

Constraints

     COMMAND: Character scalar
     WAIT: Logical scalar
     EXITSTAT: Integer scalar 
     CMDSTAT: Integer scalar 
     CMDMSG: Character scalar

Rules

     If a condition occurs that would assign a nonzero value to CMDSTAT but the CMDSTAT variable is not present, error
 termination is initiated.

Language Standard Considerations

     In Fortran 2008, we can call EXECUTE_COMMAND_LINE to execute a command line. We should flag language level errors
 and issue a message when this intrinsic is used with -qlanglvl set to anything other than 2008std, 2008pure or extended.

Messages

 If an error condition occurs:
 We assign the following message to the CMDMSG argument if it is present, otherwise, we issue the message to stderr.
 Message Text: "'filename', line line_no: 1525-xxx Error encountered while attempting to execute the command: 
 \n%1$s\n", here %1$s means command string.

Samples

     1. call EXECUTE_COMMAND_LINE('rm -rf script') 
        open(1, FILE='script') 
        call EXECUTE_COMMAND_LINE('date | awk \'{printf \" %s \", $1}\' >> script.out')
 
     2. character(len=10) :: cmd = "du -s -m"
        call EXECUTE_COMMAND_LINE(cmd, .TRUE.)
 
     3. integer :: exit = 0
        call EXECUTE_COMMAND_LINE('/bin/mv mod1.mod mod2.mod', .TRUE., exit)
 
     4. integer :: exit, cmdstat = 0
        call EXECUTE_COMMAND_LINE('echo running on = \'hostname\'', .TRUE., exit, cmdstat)
 
     5. implicit none
        character(200) :: message = ""
        integer :: exit_s = 0, cmd_s = 0
 
        call execute_command_line("mv f.1, f.2", exitstat=exit_s, cmdstat=cmd_s, cmdmsg=message)
 
        if (cmd_s .ne. 0) then
          if (cmd_s .eq. -1) then
            print *, "command execution not supported on this system"
            stop 1
          end if 
 
          print *, message
          stop 2
        end if
 
        if (exit_s .ne. 0) then
          stop 3
        end if
 
        end

Testing Considerations

     Diagnostic testing:
  1. Test that no message is emitted when EXITSTAT, CMDSTAT and CMDMSG are omitted or present
  2. Test that no message is emitted when an empty character string is given to COMMAND
  3. Test that CMDSTAT is equal to -2 when no error condition occurs and WAIT is set to .FALSE.
  4. If a condition occurs that would assign a nonzero value to CMDSTAT but the CMDSTAT variable is not present,
    error termination is initiated. We should test if the program will be terminated.
  5. Test type/rank of arguments being passed in.
     Fuctional testing:
  1. Whether the user specified command can be executed correctly and The correctness of the results need to be checked
  2. Whether EXITSTAT, CMDSTAT or CMDMSG argument can be set correctly by program according to the F2008 standard if they are specified.
High-Level Implementation Design

Implementation Design Overview

 math_tab.def & math_arg.def: 
     (1)In math_tab.def, add one new intrinsic EXECUTE_COMMAND_LINE, set relevant attribute flags and the argument 
 description, argument keyword specification.  
     (2)In math_arg.def, specify all arguments, COMMAND, WAIT, EXITSTAT, CMDSTAT, CMDMSG.
     (3)Then use mathtab tool to create math_tab.h mhashaix.h.
 
     Use runtime interface routines in Fortran to implement EXECUTE_COMMAND_LINE, and there should be interfaces that     
 accept INTEGER(2), INTEGER(4) and INTEGER(8), we write _xlhecl(), _xliecl(), _xljecl() routines in Fortran. And within
 these routines, the program will call the _xlf_exec_cmd() C routine that interfaces system(), since we might want to call
 more than "system" if we decide to catch stderr or support asynchronous execution in the future. 
 
     If system(NULL) returns zero, it indicates that a command processor is unavailable, command line execution is not supported, 
 so we'll set CMDSTAT to -1. 
 
     If a condition occurs that would assign a nonzero value to CMDSTAT but the CMDSTAT variable is not present, error 
 termination should be initiated.

Inter-Component Communication/Interfaces

     The EXECUTE_COMMAND_LINE is an intrinsic subroutine, it is implemented using BIF, it won't get inlined by FE.
 The BIF name is __EXECUTE_COMMAND_LINE, it has 5 arguments, COMMAND, WAIT if specified or provided with the default
 value .TRUE., EXITSTAT if specified or a temp scalar integer created to pass to the runtime, CMDSTAT if specified or 
 provided by __supplyoptarg, CMDMSG if specified or provided by __supplyoptarg. Since BIF __EXECUTE_COMMAND_LINE takes 
 character arguments(COMMAND & CMDMSG) and characters are not null-terminated by Fortran, ASTI needs to augment the call 
 with the length information. Therefore FE generates BIF __EXECUTE_COMMAND_LINE and then ASTI generates the call to the 
 runtime entry(subroutine) EXECUTE_COMMAND_LINE, it is implemented in C not Fortran and there are three interfaces, 
 _xlhecl, _xliecl and _xljecl, they accept INTEGER/LOGICAL(2), INTEGER/LOGICAL(4) and INTEGER/LOGICAL(8) accordingly.
 
     The routine prototype for EXECUTE_COMMAND_LINE is as follows:
     void _xlhecl(char *command, int line_no, char *filename, int16_t wait, int16_t *exitstat, int16_t *cmdstat, 
                  char *cmdmsg, int cmdstat_present, int cmdmsg_present, long command_len, long filename_len,
                  long cmdmsg_len)
 
 
     void _xliecl(char *command, int line_no, char *filename, int32_t wait, int32_t *exitstat, int32_t *cmdstat, 
                  char *cmdmsg, int cmdstat_present, int cmdmsg_present, long command_len, long filename_len,
                  long cmdmsg_len)
      
  
  
     void _xljecl(char *command, int line_no, char *filename, int64_t wait, int64_t *exitstat, int64_t *cmdstat, 
                  char *cmdmsg, int cmdstat_present, int cmdmsg_present, long command_len, long filename_len,
                  long cmdmsg_len)
