 Access Change History

Feature Overview


Fortran 2008 adds the C_SIZEOF function to the ISO_C_BINDING intrinsic module. C_SIZEOF is similar to C's sizeof operator

Key Requirements

C_SIZEOF is only accessible via using the ISO_C_BINDING intrinsic module.

Key Design Issues

1. C_SIZEOF() will add to existing intrinsic ISO_C_BINDING , it's not a runtime function, and will be folded in FE
2. C_SIZEOF() can be used only when using ISO_C_BINDING module
3. Get the type of the argument
4. Determine the size of the argument via type of argument

Key Design Decisions

1. In the ISO_C_BINDING intrinsic module add a new function C_SIZEOF() , will not add a runtime function
2. The function should not be exposed unless the user use the ISO_C_BINDING intrinsic module
3. the size can be determined by dictionary or semantic stack rather than calculating manually and hard code
4. for one special explict-shape array with non-constant bound(LR 5.3.8.2), the size will get from ASTI BIF __size


Testing Strategy

The test will cover diagnostic and functional test.

The diagnostic test will check the syntax messages at compile time when
- X is not an interoperable data entity
- X is an assumed-size array

The functional test will focus on
- X are intrinsic types with all named constants listed in table 15.2
- X is a pointer which is interoperable with C pointer
- X is an array of intrinsic types or a derived-type which is interoperable with C struct type
- X is a global variable (a variable declared in the scope of module)
- result type is of type integer(C_SIZE_T)
- Fortran code will call C procedure to get the size in bytes of an interoperable entity (returned by C sizeof operator), and then compare it with the result value of C_SIZEOF.
- C code will call Fortran procedure to get the size in bytes of an interoperable entity(returned by C_SIZEOF), and then compare it with the result value of C sizeof operator

User Documentation

Overview

Fortran 2008 adds the C_SIZEOF function to the ISO_C_BINDING intrinsic module

User Benefits

User can now use C_SIZEOF(X) to get the bytes of X , where X shall be an interoperable data entity that is not an assumed-size array.

Public Interface

C_SIZEOF(X) in intrinsic module ISO_C_BINDING

Get the size of X in bytes.

X shall be an interoperable data entity and shall not be an assumed-size array.

Result of C_SIZEOF() is scalar integer of kind C_SIZE_T .

Result Value. If X is scalar, the result value is the value that the C processor returns as the result of applying the sizeof operator to an object of interoperable with X.

If X is an array, the result value is the value that the C processor returns as the result of applying the sizeof operator to an object of interoperable with X, multiplied by the number of elements in X.


Constraints

1. Actual argument X must be interoperable type

2. Actual argument X must not be an assumed-size array

Rules

None.

Language Standard Considerations

This function conforms to the Fortran 2008 standard

Messages

1. If the argument is not interoperable data entity or argument is assumed-size array , FFE will emit a message:

1513-041 (S) Arguments of the wrong type were specified for the INTRINSIC procedure "C_SIZEOF"

2. The function is available at all langlvl settngs. this function will still work when the function is used with langlvl 2003 and below, a message will emit :

1518-256 (L) The intrinsic procedure c_sizeof (with these arguments) is not permitted by the Fortran xxx standard. (xxx = 2003 , 95 , etc)

Testing Considerations

Need to consider the following cases when implementing test cases

- Option -qlongdouble will affect the size of an entity of type long double and long double compiex in C code.

- Option -q64 will affect the result value of C_SIZEOF when X is of type C_LONG/C_SIZE_T/type(C_PTR)/type(C_FUNPTR) .

- For derived-type, there are alignment issues if components are of different types. The test will apply the -qalign options to Fortran and C code to make sure they use the same alignment mechanism.

High-Level Implementation Design

Implementation Design Overview

C_SIZEOF(X) will be folded in FFE

1. X is scalar
1.1. X is intrinsic type, C_SIZEOF(X) will be replaced by the value of corresponding kind type parameter that is defined in ISO_C_BINDING
e.g. REAL(C_DOUBLE) :: d
C_SIZEOF(d) will be replaced by 8 (see 7.4 below)

1.2. X is Type(C_PTR or C_FUNPTR), C_SIZEOF(X) will be the machine word length

1.3. X is derived type, C_SIZEOF(X) will be the sum of the bytes of each component in X and add the bytes introduced by byte alignment:
Note: derived type is interoperable with C struct when it only has data components with interoperable type and type parameter in its definition, so its size can be determined in FE
TYPE BIND(C) :: f_type
INTEGER(2)::ia(3)
INTEGER(4)::ib
INTEGER(2)::ic(3)
INTEGER(4)::id
END TYPE
TYPE(f_type) :: t_a

By default, byte alignment is 4 ,then total bytes of t_a is 3*2+2+4+3*2+2+4 = 24. C_SIZEOF(t_a) will be 24.
Between t_a%ia(3) and t_a%ib there is a padding of 2 bytes,2 bytes padding also exists between t_a%ic(3) and t_a%id,so size is 24bytes not 20bytes

2. X is array
2.1. X is whole array or array section, X only can be explicit-shape array or character array whose elements are all with length 1
2.1.1 If X has explicit bound and rank , then return value is the the size of scalar of this array multiplied the number of elements in this array
2.1.2 If An explicit-shape array that is a named local variable of a subprogram or BLOCK construct may have bounds that are not constant expressions:
function func(integer i)
integer(4)::arr(1:i)
! in this case , ASTI BIF __size will be called to determined the size, the size will be 4*i
end

3. X is character
only character of length 1 is interoperable with C
3.1 character(len=1) ch will be allowed(interoperable with C) , while character with length > 1 will not be allowed
3.2 character array , it will be allowed (interoperable with C) only when each element of this array with length 1,e.g. character array (character(len=1) ch(10))
3.3 substring with length 1 is allowed (ch(1:3) will not be allowed), for ch(i:j) , ch(i:i) is disallowed because i and j are not constant expressions
3.4 concatenation with length == 1 is allowed , otherwise is disallowed

4. X is logical type
if -qintlog is not specified then only logical(c_bool) / logical(1) is allowed, result is 1
if -qintlog is specified then then logical(2) / logical(4) / logical(8) / logical is also allowed , results is 2 / 4 / 8 / size of defualt logical type

5. X is expression or function reference , result will be determined by its type and type parameter .

6. X is constant or subobject of a constant.
6.1 numeral
6.1.1 literal,return value is the corresponding length of its default type and type parameter. e.g. c_sizeof(0) == 4 , c_sizeof(4.0) == 4, c_sizeof(4.0e0) == 4, c_sizeof(4.0d0) == 8
6.1.2 defined with "parameter" , return value is the corresponding length of its type and type parameter
6.2 character
only character of length 1 is interoperable with C
6.2.1 literal, return value is the actual length of it. e.g. c_sizeof("a") == 1 and c_sizeof("ab") is invaild
6.2.2 defined with "parameter". e.g. character(len=1) ,parameter :: charr="1", c_sizeof(charr) == 1, length > 1 will be not allowed
6.2.3 the same as 3.3
6.2.4 the same as 3.4

7. Key Code
7.1 math_tab.def & math_arg.def:
7.1.1 In math_tab.def, add a new intrinsic function __c_sizeof (then rename to c_sizeof via iso_c_binding.F see 7.2 below)
7.1.2 Use mathtab tool to create math_tab.h and mhashaix.h.
7.2 iso_c_binding.F
add iso_c_binding_proc_impl module with intrinsic __c_sizeof , then rename __c_sizeof to c_sizeof in ISO_C_BINDING module , so that , c_sizeof only can be used via iso_c_binding

add new function(named checkcsizeofarg() ) to check the parameters of c_sizeof() , diagnostic code is here.
