Overview

POPCNT and POPPAR are new intrinsics added in the Fortran 2008 standard. XL Fortran already supports these two intrinsics as IBM extensions. Currently the functions are evaluated during run time, and are not allowed as constant expressions. To fully comply with the F2008 standard, this feature allows the two functions to be used in constant expressions.

Syntax:

POPCNT (I)

Return number of one bits. I shall be of type integer. Ex. POPCNT(5) = 2

POPPAR (I)

Return parity expressed as 0 or 1. I shall be of type integer. Ex. POPPAR(5) = 0

The key requirement is to allow the functions in constant expressions. The compiler no longer issues an error message when the functions are used as constant expressions.

Test Strategy

The test strategy will be based on the checking of new features of the POPCNT and POPPAR functions. In particular, the test cases will cover such features as using this functions in constant expression and supporting only an integer type of argument in F2008 standard.

Test cases will be divided on the functional and diagnostic parts.

Diagnostic tests will check that the compiler will continue to issue error message if langlvl lower than F2008 standard (feature: constant expression), and will issue error message if argument is not an integer type (in F2008 standard). To reach success result of testing, we will use different combination of -qlanglvl parameter to cover all error message issues.

Functional test cases will check the feature of constant expression. We will test of ability of POPCNT and POPPAR to be used in an initialization expression or specification expression. We will consider this intrinsic in different program parts such as subroutine, function and modules. Also TCs will cover array features as argument in POPCNT and POPPAR functions.
