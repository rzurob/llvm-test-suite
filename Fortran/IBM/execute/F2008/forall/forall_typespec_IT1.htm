Overview

The Fortran 2008 standard allows optional type specifications for index variables in a FORALL statement/construct, for which this feature will add support.

With this extension, users can include a type specification in the FORALL header for the corresponding index variables. The type specification must be an integer type and applies to all indices in the FORALL header.

This feature is a Fortran 2008 specific extension to FORALL. The type specifier in the FORALL header will be flagged by a language level message when language level F2003 or lower is in effect.

Following is the FORALL syntax, including the optional type-spec:

  FORALL ( [ type_spec :: ] forall_triplet_spec_list [, scalar_mask_expr ] )

Here is an example of its use:

        implicit none
        integer, parameter :: LIMIT = 255
        integer(1) :: flag(LIMIT)
        flag = 0
        FORALL ( integer(2) ::  i = 2 : LIMIT : 2)
           flag(i) = 1
        END FORALL
        print *, flag(LIMIT)
        END

Test Strategy


The testing will be split into two groups, diagnostic and functional.

Diagnostic tests will verify that appropriate error messages are emitted when forall is given an invalid type specifier, such as a non-integer type. Various other invalid uses are expected to generate suitable error messages. A language level message is expected when a forall declaration is correctly used but with an invalid language level in effect.

Functional tests will include basic valid use cases as well as nesting of the forall construct. It can be nested within itself, with do, omp, SE, TM, or in other constructs. If general, where a forall loop was valid before the extension, it should be verified that the forall loop is still valid using the extension and an appropriate type specifier.

Test Procedure and Scenario


There will be diagnostic test cases to verify that error messages are emitted in the following situations:

- a non-integer type specifier is used in the forall header
- multiple type specifiers are used in the forall header
- a language level message is emitted, with a valid use case, when the specified language level is not 2008STD.

There will be one or more test cases to check the following:

- nesting of the forall construct in other constructs, such as do loops, omp do, SE or TM regions, smp, blocks, etc
- validate basic use of the forall extension where forall was used before.  A subset of existing test cases in the hpf_forall bucket will be ported in addition to the new test case material generated.
- that external scope variable names reused in the forall header do not result in those variables being modified

Test coverage of the following is also scattered among the test cases:
- both forms of integer declarations in the type specifier: integer*n and integer(n), with n = 1, 2, 4, 8
- nesting FORALL constructs/statements with different type specifiers in their headers
- using FORALL constructs/statements with different lettering case

The hpf_forall bucket provides much of this coverage with the forall construct and assignment before the extension.  A selected subset of these test cases will be ported for functional testing of this feature.
