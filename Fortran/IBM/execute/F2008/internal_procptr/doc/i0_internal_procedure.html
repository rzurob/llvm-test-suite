Access Change History

 Access Change History

Feature Overview

Fortran 2008 allows internal procedures and pointers to such procedures to be actual arguments.

Key Requirements

Internal procedures and pointers to internal procedures are allowed to be actual arguments, 
and procedure pointers that point to internal procedures are also allowed, they are both complied with
the Fortran 2008 standard.

Key Design Issues

(1)There is a special XLF optimization that changes that calling convention of internal procedures
with intent(in) arguments from by-reference to by-value. This optimization is only valid when the
internal procedure is not an actual argument and not pointed to by a procedure pointer. Now we can
take an internal procedure as an actual argument, then the result is unexpected if the optimization
is in effect.

(2)Front-end debug option "internalactual"'s behavior.

(3)As F2008 standard suggests, an internal procedure must not be invoked using a procedure pointer after the host
instance completes execution.

Key Design Decisions

(1)We disable the optimization all the time, but hide the optimization behind the debug option: IPROCINOPT, indicating that
we do optimizations for intent(in) arguments of internal procedures by changing the calling convention from by-reference to
by-value.     

(2)The debug option "internalactual" is used to support internal procedures as actual arguments. We don't need it now.

(3)If an internal procedure is invoked using a procedure pointer, and at that time the host instance has completed execution,
the procedure pointer will become undefined, it's a dangling pointer and when running the program, we may get core dump.
At the compile time the problem cannot be checked, so it's the user's responsibility to maintain the program, we won't do anything
related to the issue. 

Testing Strategy

Unit Testing:
- Functional:
(1)Test that when the internal procedures(functions, subroutines, with no args, intent(in) args,
intent(out) args, etc) are passed as actual arguments of another procedure, we can get the expected
result if calling this procedure.

(2)Test that when the pointers to internal procedures(functions, subroutines, with no args, intent(in) args,
intent(out) args, etc) are passed as actual arguments of another procedure, we can get the expected result
if calling this procedure.

- Diagnostic:
(1)Test that when the internal procedures or the pointers to such procedures are passed as actual arguments,
and -qlanglvl=<older standards> is specified, a langlvl message(L) is issued.

(2)Test that when the internal procedures are the targets of procedure pointer assignment, and -qlanglvl=<older standards>
is specified, a langlvl message(L) is issued.

- The feature only involves the syntactic and semantic change, it won't have any appreciable impact on performance,
then high optimization option testing is unnecessary.


 

User Documentation

Overview
Fortran 2008 allows internal procedures and pointers to such procedures to be actual arguments. 
Fortran 2008 also allows procedure pointers to point to internal procedures.

User Benefits

Users now can pass an internal procedure or pointer to internal procedure as the actual argument
of other procedures.

Public Interface

None.

Constraints

None.

Rules

An internal procedure must not be invoked using a procedure pointer after the host instance completes execution.

Language Standard Considerations

(1)When internal procedures or pointers to such procedures have been passed as actual arguments
of other procedures, we need to issue a langlvl message if -qlanglvl=2003std or below is specified.

(2)When internal procedures are the targets of procedure pointer assignment
, we also need to issue a langlvl message if -qlanglvl=2003std or below is specified.

Messages
When internal procedures are the targets of procedure pointer assignment, and  -qlanglvl=2003std or below is
specified, we issue the following langlvl message:
"(L) An internal procedure is not permitted as the target of a procedure pointer assignment statement by the %1$s standard".

Samples

 
1. Internal procedure passed as actual argument to one module procedure:

module m 
  contains
    subroutine sub(print) ! Module procedure.
      interface
        subroutine print(arg1, arg2, arg3, arg4, arg5)
          integer, intent(in) :: arg1
          real, intent(in) :: arg2
          complex, intent(in) :: arg3
          character(*), intent(in) :: arg4
          logical, intent(in) :: arg5
        end subroutine
      end interface

      real :: r = 3.0

      ! Call internal procedure 'print' which is passed as actual argument here.
      call print(1 + 2, r * r - r, (2, 3), "hello world", .TRUE.) 
    end
end module        

program PassInterProc
   use m

   call sub(print)

   contains

     ! Internal procedure 'print' to be passed as actual argument
     subroutine print(arg1, arg2, arg3, arg4, arg5) 
       integer, intent(in) :: arg1
       real, intent(in) :: arg2
       complex, intent(in) :: arg3
       character(*), intent(in) :: arg4
       logical, intent(in) :: arg5

       print *, "print: integer =", arg1
       print "(a, f3.1)", " print: real = ", arg2
       print "(a, f3.1, f3.1)", " print: complex = ", arg3
       print *, "print: characters = ", trim(arg4)
       print *, "print: logical =", arg5
     end subroutine 
end program

2. Pointer to internal procedure which is passed as actual argument to one module procedure:

module m 
  contains
    subroutine sub(print) ! Module procedure.
      interface
        subroutine print(arg1, arg2, arg3, arg4, arg5)
          integer, intent(in) :: arg1
          real, intent(in) :: arg2
          complex, intent(in) :: arg3
          character(*), intent(in) :: arg4
          logical, intent(in) :: arg5
        end subroutine
      end interface

      real :: r = 3.0

     ! Call internal procedure 'print' which is passed as actual argument here.
      call print(1 + 2, r * r - r, (2, 3), "hello world", .TRUE.)
    end
end module        

program PassInterProcPtr
   use m

   interface
     subroutine print_interf(arg1, arg2, arg3, arg4, arg5) 
       integer, intent(in) :: arg1
       real, intent(in) :: arg2
       complex, intent(in) :: arg3
       character(*), intent(in) :: arg4
       logical, intent(in) :: arg5
     end subroutine
   end interface
 
   procedure(print_interf), pointer :: p ! Procedure pointer.
   p => print

   ! We pass procedure pointer which points to internal procedure as the actual argument. 
   call sub(p) 

   contains
     subroutine print(arg1, arg2, arg3, arg4, arg5) 
       integer, intent(in) :: arg1
       real, intent(in) :: arg2
       complex, intent(in) :: arg3
       character(*), intent(in) :: arg4
       logical, intent(in) :: arg5

       print *, "print: integer =", arg1
       print "(a, f3.1)", " print: real = ", arg2
       print "(a, f3.1, f3.1)", " print: complex = ", arg3
       print *, "print: characters = ", trim(arg4)
       print *, "print: logical =", arg5
     end subroutine 
end program

3. An internal procedure that accesses host variables and that is invoked via function pointer 
from outside the host procedure scoping unit.

module m
  real, parameter :: PI = 3.14
  integer, dimension(5) :: arr = [1,2,3,4,5]
  character(10) :: string = "HelloWorld"

  type :: base(k, l)
    integer, kind :: k
    integer, len :: l
    integer(k) :: value = 999
    character(l) :: str = "Hello"
  end type

  type(base(4,5)), save :: dt
end module

program p
  call wrapper()
end program

subroutine wrapper()
  use m

  interface
    subroutine call_sub(proc)
      interface
        subroutine proc()
        end subroutine
      end interface
    end subroutine
  end interface

  call call_sub(sub)

  contains
    subroutine sub() ! Internal procedure sub
      print "(a, f4.2)", "PI is ", PI
      print *, "string = ", string
      print *, "arr = ", arr
      print *, "base = ", dt
    end
end subroutine

subroutine call_sub(proc)
  interface
    subroutine proc()
    end subroutine
  end interface

  procedure(proc), pointer :: ptr

  ptr=>proc

  print *, "Start to call: "
  call ptr()  ! Call by internal procedure pointer
end

High-Level Implementation Design

Implementation Design Overview

(1)We allow internal procedures and pointers to such procedures to be passed as actual arguments,
then it is necessary to remove the relevant check and don't issue error messages like what xlf does before.

(2)We disable all the optimizations(Mentioned in "Key Design Issues" & "Key Design Decisions" sections)
that will change the calling convention from call-by-ref to call-by-val for the internal procedures with intent(in) arguments.
But we hide the optimizations behind the new debug option "IPROCINOPT".

(3)Remove the debug option "internalactual".

(4)Issue langlvl(L) message if appropriate(Mentioned in "Language Standard Consideration" & "Messages" section).
