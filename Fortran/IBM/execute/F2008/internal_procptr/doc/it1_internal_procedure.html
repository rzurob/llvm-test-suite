Overview

Fortran 2008 allows an internal procedure as actual argument or procedure pointer target. This test plan covers typical scenarios based on our understanding on the standard.



Test Strategy

The test will be focusing on the following areas:

- Pass internal procedures as actual argument and test argument association
- Associate procedure pointer to Internal procedure
- langlvl test
- Host instance. Make sure the host instance is correctly accessed
- Associate procedure pointer component with internal procedure
- Miscellaneous testing


All tests will primarily target at making sure that

-- The internal procedure is correctly associated, and

-- The entities in its host instance can be correctly accessed.


A typical example:


  PROGRAM intproc_basic_1
  INTEGER :: i=0

  CALL Intsubp(Intsub1)
  IF ( i .NE. -1 ) STOP 11

  CONTAINS

  SUBROUTINE Intsub1()
    i = -1
  END SUBROUTINE

  SUBROUTINE Intsubp(proc)
  PROCEDURE() :: proc
    CALL proc()
  END SUBROUTINE

  END
