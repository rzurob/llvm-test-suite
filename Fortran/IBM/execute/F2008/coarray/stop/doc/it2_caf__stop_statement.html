Diagnostic Tests:

 

Scenario name

 

Type of test / Comments

stop_d001.scenario

-qcaf

Test that the error message from the STOP statement will be printed out on stderr (not stdout) and will be prefixed with "STOP".

stop_d002.scenario

-qnocaf

Same as above

stop_d011.scenario

-qcaf

Test that when errimagenum=yes is in effect, the image number of the executing image precedes STOP message.

stop_d012.scenario

-qcaf

Test that when errimagenum=no is in effect, the image number of the executing image does not precedes STOP message.

stop_d013.scenario

-qcaf

Test that when errimagenum is not specified (default to ‘no’), the image number of the executing image does not precedes STOP message.

stop_d015.scenario

-qnocaf

Test that errimagenum=yes has no effect on sequential (non-CAF) mode.

stop_d016.scenario

-qnocaf

Test that errimagenum=no has no effect on sequential (non-CAF) mode.

stop_d017.scenario

-qnocaf

Test that default errimagenum value has no effect on sequential (non-CAF) mode.

stop_d021.scenario

-qcaf

Test that when stop-code specified for the STOP statement is not ‘0’, the runtime will replace it with ‘0’ as the program exit status and issue warning message:

1525-705 The %C1 statement with stop code %I1 will terminate the program with return code %I2.

stop_d022.scenario

-qcaf

Similar as above

With “errimagenum=yes”

stop_d025.scenario

-qcaf

Test that a pure subprogram must not contain an STOP statement. Error message is issued:

1516-108 (S) A pure subprogram must not contain a STOP or a PAUSE statement.

stop_d026.scenario

-qnocaf

Similar as above

stop_d031.scenario

-qcaf

If a statement label is specified in a DO statement, STOP must not be used to terminate that DO construct.

stop_d032.scenario

-qnocaf

Similar as above

stop_d041.scenario

-qcaf

Test that the error messages that are issued when the STOP stop-code is not scalar constant expression of character or integer type.

stop_d042.scenario

-qnocaf

Similar as above

 

 

Functional Tests:

 

Scenario name

 

Type of test / Comments

stop_f001.scenario

-qcaf

Test that due to the limitation of POE, whatever the stop-code is, the value 0 is always supplied as the process exit status for the STOP statement. Also looks for the corresponding STOP message.

Stop values: 0, 2-127

stop_f002.scenario

-qcaf

Same as above

Stop values: 1, 128-255

stop_f003.scenario

-qnocaf

Test that on sequential (non-CAF) mode, program exit status is the same as the supplied stop-code value in the STOP statement. Also looks for the corresponding STOP message.

Stop values: 0, 2-127

stop_f004.scenario

-qnocaf

Same as above

Stop values: 1, 128-255

stop_f005.scenario

-qcaf

Test that when user does not explicitly specify any stop-code for STOP (i.e. default one), then the program exit status is 0

Also looks for "STOP" prefix in the output.

stop_f006.scenario

-qnocaf

Similar as above

Since a character constant or digit string is not specified, the keyword STOP is not printed.

stop_f007.scenario

-qcaf

Test that when explicitly specify string (non integer) value for STOP, then the program exit status is 0

Also looks for "STOP" prefix in the output.

stop_f008.scenario

-qnocaf

Same as above

stop_f020.scenario

-qcaf

Testing the STOP statement initiating normal termination of execution - main program

stop_f021.scenario

-qcaf

Similar as above           internal subroutine

stop_f022.scenario

-qcaf

Similar as above           internal function

stop_f023.scenario

-qcaf

Similar as above           external subroutine

stop_f024.scenario

-qcaf

Similar as above           external function

stop_f025.scenario

-qcaf

Similar as above           module subroutine

stop_f026.scenario

-qcaf

Similar as above           module function

stop_f040.scenario

-qcaf

STOP initiating normal termination of execution of ordered image work. All images complete in desired order.

stop_f041.scenario

-qcaf

STOP initiating normal termination of circular shift

stop_f042.scenario

-qcaf

STOP initiating normal termination of execution and does NOT interrupt others images work in the middle
