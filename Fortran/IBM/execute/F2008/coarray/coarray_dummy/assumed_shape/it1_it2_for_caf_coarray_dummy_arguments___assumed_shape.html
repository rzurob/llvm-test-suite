Overview

%Note{Simple description of the feature; In which product it is first implemented, is it common for other platforms, is it common for both C and C++ and so on.}%

There is no I0 for this feature. This feature tests the assumed shape dummy argument co-arrays. An example of a co-array declaration:

       real, save :: a(N,N)[p,*]

This is a real array of size N*N which will be replicated on each image. If p = 4 and assuming there are 16 images then the CAF environment will create a map as follows:

1 2 3 4
1 1 5 9 13
2 2 6 10 14
3 3 7 11 15
4 4 8 12 16

In this map the image 12 will be referred to as (/ 4,3 /) and so on. It's easy to access such a co-array if the number of images (already known) is a square but becomes difficult if this number is dynamic or is not a square. An example code is given in Section Test Strategy to make this more clear.

Test Strategy

%Note{Test strategy used in the test}%

The purpose of this testing is to test the use of assumed shape dummy arguments co-arrays in the CAF environment. All the tests are functional tests. Following are some of the considerations for writing the TCs:

  • Different shapes and ranks of the array and the co-array will be tested.
  • Values will be set in one image, copied to other images and the copy will be verified.
  • Passing of co-array with different shapes and ranks to other intrinsic functions.
  • Use of co-arrays in the derived type will be tested with different shapes and ranks.
  • As mentioned above, in Section Overview, if the number of images is a square (in above case 4*4 = 16) it's easy to access the co-array. But If the number of images is dynamic it's difficult to map the images onto the co-array. Here is an example:
    • We want to set the data in image 1 and pass it to all other images

                   integer :: x, y, row
                   integer, save :: A(3)[3,*]

                   if (this_image() .EQ. 1) then
                      do x = 1, 3
                         do y = 1, 3
                            do row = 1, 3
                               A(row)[x,y] = row + this_image() - 1            ! All the images to be initialized with values: 1, 2 and 3
                            end do
                         end do
                      end do
                   end if

                   SYNC ALL

    • The above code assumes the co-array is 3 x 3. But what if the above code is run for 1 or 2 image(s). The loop will try to access an image (3,1) which is not present. We need to introduce some checks so that the above code should run good with any (dynamic) number of images. See the three last TCs as listed in the table below for these checks.
