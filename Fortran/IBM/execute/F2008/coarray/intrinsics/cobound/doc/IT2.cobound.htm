UNIT TESTS

Unit tests were created for this feature, so I devised my test plan utilizing them as best as possible to avoid duplication. Here is a list of major coverage points in the Unit test bucket that I've omitted from my test plan:

Diagnostic

  • Check lcobound/ucobound second argument accepts only valid values (i.e. 1 <= DIM <= corank).
  • Check lcobound/ucobound third argument accepts only valid values (i.e. 1, 2, 4, 8).
  • Test misspelled argument tag names.
  • Arguments to lcobound/ucobound cannot have optional parameter.
  • Constant expressions using lcobound/ucobound must be known at compile time.

 Functional
  • Test lcobound/ucobound kind types with -qintsize.
  • Test lcobound/ucobound in constant expressions.
