Overview


Test Strategy

  • -qcaf will be a required xopt for all scenarios unless otherwise stated.
  • -qcaf=images and -qcaf=dnodes will only be specified for test cases where the image count has a direct impact on the test cases' results.
  • Test cases will be designed to be platform independent.
Diagnostic testing will consist of argument and return type testing of lcobound() and ucobound() intrinsics.

Functional testing will consist of testing lcobound() and ucobound() with different parameter values, different coarray sizes, and different image sizes to see that the results are produced correctly and consistently.

Unit testing was completed beforehand and will be leveraged to improve overall coverage and avoid duplication.

UNIT TESTS

Unit tests were created for this feature and so I've tried to devise my test plan utilizing those test cases as best as possible to avoid duplication. Here is a list of major coverage points in the Unit test bucket that I've omitted from my test plan:

Diagnostic

  • Check lcobound/ucobound second argument accepts only valid values (i.e. 1 <= DIM <= corank).
  • Check lcobound/ucobound third argument accepts only valid values (i.e. 1, 2, 4, 8).
  • Test misspelled argument tag names.
  • Arguments to lcobound/ucobound cannot have optional parameter.
  • Constant expressions using lcobound/ucobound must be known at compile time.

Functional
  • Test lcobound/ucobound kind types with -qintsize.
  • Test lcobound/ucobound in constant expressions.
