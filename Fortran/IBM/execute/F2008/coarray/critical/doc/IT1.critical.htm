Overview

The Critical Construct is defined as:

Critical

block

End Critical

It generates a critical region, whereby only one image can execute it at a time. Note the order of execution cannot be controlled by Critical. This feature is similar to Critical constructs in other standards (ex. OMP) but this is directly embedded into Fortran and applies to synchronizing images rather than threads.

Test Strategy

The following are some items covered throughout the test suite but may be explicitly stated in the plan below.
  • Most of the testcases will test coarrays in the critical section local to the compilation use, but others will explicitly be stated to test global and dummy argument coarrays.
  • Array and scalar type coarrays used in a critical construct will vary throughout the testcases.
  • Test different values for corank and cobounds used throughout.
  • The critical constructs will be tested in all compilation units.

 The main strategy is to test that writing to the same coindex object by multiple images occurs one at a time. A variety of methods will be used to test this, but they follow a similar to:
 critical
 count[1] = count[1] + 1
 end critical
