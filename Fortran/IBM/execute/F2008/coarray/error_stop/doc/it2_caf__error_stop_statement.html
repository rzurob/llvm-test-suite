Diagnostic Tests:

 

 Scenario name

 

 Type of test / Comments

error_stop_001d.scenario

-qcaf

Test that the error message from the ERROR STOP statement will be printed out on stderr (not stdout) and will be prefixed with "ERROR STOP".

error_stop_002d.scenario

-qnocaf

Same as above

error_stop_011d.scenario

-qcaf

Test that when errimagenum=yes is in effect, the image number of the executing image precedes ERROR STOP message.

error_stop_012d.scenario

-qcaf

Test that when errimagenum=no is in effect, the image number of the executing image does not precedes ERROR STOP message.

error_stop_013d.scenario

-qcaf

Test that when errimagenum is not specified (default to ‘no’), the image number of the executing image does not precedes ERROR STOP message.

error_stop_015d.scenario

-qnocaf

Test that errimagenum=yes has no effect on sequential (non-CAF) mode.

error_stop_016d.scenario

-qnocaf

Test that errimagenum=no has no effect on sequential (non-CAF) mode.

error_stop_017d.scenario

-qnocaf

Test that default errimagenum value has no effect on sequential (non-CAF) mode.

error_stop_021d.scenario

-qcaf

Test that when stop-code specified for the ERROR STOP statement is not 1, the runtime will replace it with one as the program exit status and issue warning message:

1525-705 The %C1 statement with stop code %I1 will terminate the program with return code %I2.

error_stop_022d.scenario

-qcaf

Similar as above

With “errimagenum=yes”

error_stop_025d.scenario

-qcaf

Test that a pure subprogram must not contain an ERROR STOP statement. Error message is issued:

1516-333 (S) A pure subprogram must not contain the ERROR STOP statement.

error_stop_026d.scenario

-qnocaf

Similar as above

error_stop_031d.scenario

-qcaf

If a statement label is specified in a DO statement, ERROR STOP must not be used to terminate that DO construct.

error_stop_032d.scenario

-qnocaf

Similar as above

error_stop_041d.scenario

-qcaf

Test that the error messages that are issued when the ERROR STOP stop-code is not scalar constant expression of character or integer type.

error_stop_042d.scenario

-qnocaf

Similar as above

 

Functional Tests:

 Scenario name

 

 Type of test / Comments

error_stop_001f.scenario

-qcaf

Test that due to the limitation of POE, whatever the stop-code is, the value 1 is always supplied as the process exit status for the ERROR STOP statement.

ES values: 0, 2-127

error_stop_002f.scenario

-qcaf

Same as above

ES values: 1, 128-255

error_stop_003f.scenario

-qnocaf

Test that on sequential (non-CAF) mode, program exit status is the same as the supplied stop-code value in the ERROR STOP statement.

ES values: 0, 2-127

error_stop_004f.scenario

-qnocaf

Same as above

ES values: 1, 128-255

error_stop_005f.scenario

-qcaf

Test that when user does not explicitly specify any stop-code for ERROR STOP (i.e. default one), then the program exit status is 1

Also looks for "ERROR STOP" prefix in the output.

error_stop_006f.scenario

-qnocaf

Similar as above

Since a character constant or digit string is not specified, the keyword ERROR STOP is not printed.

error_stop_007f.scenario

-qcaf

Test that user explicitly specify string (non integer) value for ERROR STOP, then the program exit status is 1

Also looks for "ERROR STOP" prefix in the output

error_stop_008f.scenario

-qnocaf

Same as above

error_stop_020f.scenario

-qcaf

Testing the ERROR STOP statement interrupting the work - main program

error_stop_021f.scenario

-qcaf

error_stop_022f.scenario

-qcaf

error_stop_023f.scenario

-qcaf

error_stop_024f.scenario

-qcaf

Similar as above           external function

error_stop_025f.scenario

-qcaf

Similar as above           module subroutine

error_stop_026f.scenario

-qcaf

Similar as above           module function
