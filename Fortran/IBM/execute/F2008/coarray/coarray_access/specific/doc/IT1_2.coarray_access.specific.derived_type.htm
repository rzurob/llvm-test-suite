Overview

Test Strategy

This feature covers similar testing to PART 1, but focuses on Derived Type coarray variables. There are two cases of coarray access testing:

Assignment:

Test assigning to and from Derived Type coarray variables, i.e., all tests are of the form "x = y", "y = x", "x%a = y", "y = x%a", "x%a = y%a", or "y%a = x%a", where x is a derived type coarray variable, y is something else, and a is the derived type component. The nature of the other variable will vary between:

  • variables vs. constants
  • storage location: stack, common, heap
  • scope: local, use associated, host associated
  • intrinsic type vs. derived type component of intrinsic type
  • scalar vs. array
  • allocatable & pointer vs. non-allocatable non-pointer
  • allocatable and pointer components is not yet supported, so this testing will be left out
Examples of each combination should be present. Not all intrinsic types need to be represented in each of the above tests, but each intrinsic type should be present in multiple tests.

Procedure calls:

Testing invocations of procedures on coarray variables, i.e., all tests are of the form "... = fun(x)" or "call sub(x)", where x is a Derived Type coarray variable. Since three features already exist testing coarray array dummy arguments, testing of them here will be avoided for the most part.
