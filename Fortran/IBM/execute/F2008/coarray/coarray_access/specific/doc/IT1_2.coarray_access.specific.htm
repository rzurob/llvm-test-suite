Overview

This feature tests non-coindexed coarray access. It is like a merging of the two "coindex object" features ("assignment statement" + "procedure calls"), except that coindexing is not involved.

Test Strategy

There will be two subclasses of tests, like the "coindex object" feature: assignment and procedure calls.

Assignment:

In "assignment", we assign to and from coarray variables, with no additional expressions, i.e., all tests are of the form "x=y", or "y=x", where x is a coarray variable and y is something else. The nature of that "something else" can be quite varied, invoking different mechanisms (the categories below are not completely orthogonal):

  • variables vs. constants
  • storage location: stack variables, common variables, module variables, objects in heap
  • scope/association: local, use associated, argument associated, construct associated, host associated (here, context matters, since the host could be the main program, an external procedure, or a module procedure)
  • intrinsic type vs. derived type component of intrinsic type
  • scalar vs. array
  • allocatable vs. pointer vs. nonallocatable nonpointer
Examples of each combination should be present. Not all intrinsic types need to be represented in each of the above tests, but each intrinsic type should be present in multiple tests.

Procedure calls:

In "procedure", we simply invoke procedures (functions and subroutines) on coarray variables, i.e., all tests are of the form "...=fun(x)" or "call sub(x)", where the actual argument x is a coarray variable. Here we are interested in the correct passing of arguments and possible assignments to effective arguments which are coarray variables. Since at least two features already exist which test coarray array dummy arguments, tests of coarray array dummy arguments will be minimal. The dummy argument may or may not itself be a coarray, and may or may not have other attributes, such as VALUE, INTENT, and OPTIONAL. If the dummy argument is a coarray, then corank and codimensions may be different than the actual argument.

Function result characteristics are of interest, as well. The more interesting ones are likely to include arrays, allocatables, pointers to allocated objects, and derived types constructed from values in coarray variables, including type parameters.

Finally, procedures may be defined internally, externally, or in modules, and may or may not be type-bound (here we will not test derived type coarray variables with type-bound procedures, but we should probably verify that intrinsic coarray variables are correctly passed to type-bound procedures associated with a non-coarray object).

Test Procedures/Scenarios

All tests below will be written to internally verify correct assignments, function return codes and procedure argument results.

Assignment
Name Description
assignment/csSimpleInteger assign INTEGER noncoarray variables to coarray variables (scalar and array) in main program and vice-versa
assignment/csSimpleLogical ditto, but LOGICAL
assignment/csSimpleCharacter ditto, but CHARACTER
assignment/csSimpleReal ditto, but REAL
assignment/csSimpleComplex ditto, but COMPLEX
assignment/csAssignConstant assign constants of various intrinsic types to coarray variables (scalar and array) in main program
assignment/csCommon assign values in variables in common to coarray variables and vice-versa
assignment/csAssociate in ASSOCIATE construct, assign associate-name to coarray variables and vice-versa
assignment/csSelectType in SELECT TYPE construct, assign associate-name to coarray variables and vice-versa
assignment/csMainHost in procedures internal to main program, assign host-associated constants and variables of various intrinsic types to host-associated coarray variables and assign coarray variables to host-associated variables
assignment/csModuleHost ditto, but procedures internal to module procedure
assignment/csModuleVariable in procedures internal to module procedure, assign to and query module coarray variables
assignment/csExternalHost ditto, but procedures internal to external procedure
assignment/csExternal in external procedure, assign constants and variables of various intrinsic types to coarray variables and assign coarray variables to variables, all of which are defined in the external procedure
assignment/csCharacterComponent in main program, assign character component of derived type variables to coarray variables and vice-versa
assignment/csIntegerComponent in main program, assign integer component of derived type variables to coarray variables and vice-versa
assignment/csAllocatableIntrinsic in main program, assign values in allocatable intrinsic variables to coarray variables and vice-versa
assignment/csPointerIntrinsic in main program, assign pointer intrinsic variables to coarray variables and vice-versa
assignment/csAllocatableDT in main program, assign intrinsic components of allocatable derived type variables to coarray variables and vice-versa
assignment/csAllocatableComponent in main program, assign allocatable intrinsic components of derived type variables to coarray variables and vice-versa

Name Description

procedure_call/csLogicalArg ditto, but LOGICAL
procedure_call/csCharacterArg ditto, but CHARACTER
procedure_call/csRealArg ditto, but REAL
procedure_call/csComplexArg ditto, but COMPLEX
