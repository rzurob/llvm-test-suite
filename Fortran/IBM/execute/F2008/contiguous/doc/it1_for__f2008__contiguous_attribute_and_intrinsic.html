Overview

One of the extensions in Fortran 2008 to Fortran 2003 is the CONTIGUOUS attribute which provides a means for the program to specify restrictions on the storage layout of pointer targets and assumed-shape dummy arguments. The feature will test CONTIGUOUS attribute, CONTIGUOUS statement and IS_CONTIGUOUS intrinsic function.

F2008 language standard defines coniguous array as an array having array elements in order that are not seperated by other data objects.

The following Syntax are related to CONTIGUOUS attribute, CONTIGUOUS statement and IS_CONTIGUOUS intrinsic

R437 component-attr-spec is ...
                                           Or CONTIGUOUS
                                           Or ...
R502 attr-spec is .....
                                           Or CONTIGUOUS
                                           Or .....

R533 contiguous-stmt is
                                           CONTIGUOUS [ :: ] object-name-list

 IS_CONTIGUOUS(ARRAY)
            - ARRAY may be of any type of an array. If it is a pointer, it shall be associated.
            - Result type is default logical scalar. It has value true if ARRAY is contiguous, and false otherwise.

The following sections in standard are related to the feature.

section 5.3.7  CONTIGUOUS attribute

- The CONTIGUOUS attribute specifies that an assumed-shape array can only be argument associated with a contiguous effective argument, or that an array pointer can only be pointer associated with a contiguous target.
-  An object is contiguous if it is
 (1) an object with the CONTIGUOUS attribute,
 (2) a nonpointer whole array that is not assumed-shape,
 (3) an assumed-shape array that is argument associated with an array that is contiguous,
 (4) an array allocated by an ALLOCATE statement,
 (5) a pointer associated with a contiguous target, or
 (6) a nonzero-sized array section (6.5.3) provided that
            (a) its base object is contiguous,
            (b) it does not have a vector subscript,
            (c) the elements of the section, in array element order, are a subset of the base object elements that are consecutive in array element order,
            (d) if the array is of type character and a substring-range appears, the substring-range species all of the characters of the parent-string (6.4.1),
            (e) only its final part-ref has nonzero rank, and
            (f) it is not the real or imaginary part (6.4.4) of an array of type complex.

-  An object is not contiguous if it is an array subobject, and
            (a)  the object has two or more elements,
            (b)  the elements of the object in array element order are not consecutive in the elements of the base object,
            (c)  the object is not of type character with length zero, and
            (d)  the object is not of a derived type that has no ultimate components other than zero-sized arrays and characters with length zero.

section 6.5.4  Simply contiguous array designators

- A section-subscript-list species a simply contiguous section if and only if it does not have a vector subscript and
             (a) all but the last subscript-triplet is a colon,
             (b) the last subscript-triplet does not have a stride, and
             (c) no subscript-triplet is preceded by a section-subscript that is a subscript.

- An array designator is simply contiguous if and only if it is
             (a) an object-name that has the CONTIGUOUS attribute,
             (b) an object-name that is not a pointer or assumed-shape,
             (c) a structure-component whose final part-name is an array and that either has the CONTIGUOUS attribute or is not a pointer, or
             (d) an array section
                       -  that is not a complex-part-designator,
                       -  that does not have a substring-range,
                       -  whose final part-ref has nonzero rank,
                       -  whose rightmost part-name has the CONTIGUOUS attribute or is neither assumed-shape nor a pointer, and 
                       - which either does not have a section-subscript-list, or has a section-subscript-list which species a simply contiguous section.

- An array variable is simply contiguous if and only if it is a simply contiguous array designator or a reference to a function that returns a pointer with the CONTIGUOUS attribute.

Section 7.2.2.3.7   Data pointer assignment
If the pointer object has the CONTIGUOUS attribute, the pointer target shall be contiguous.

Section 7.2.2.3.9
If bounds-remapping-list is specified, the pointer target shall be simply contiguous (6.5.4) or of rank one. It shall not be a disassociated or undefined pointer, and the size of the pointer target shall not be less than the size of the pointer object. The elements of the target of the pointer object, in array element order (6.5.3.2), are the first SIZE (data-pointer-object) elements of the pointer target.

Section 12.5.2.4.9  Ordinary dummy variables
If the dummy argument has the TARGET attribute, does not have the VALUE attribute, and either the effective argument is simply contiguous or the dummy argument is a scalar or an assumed-shape array that does not have the CONTIGUOUS attribute, and the effective argument has the TARGET attribute but is not a coindexed object or an array section with a vector subscript then
             any pointers associated with the effective argument become associated with the corresponding dummy argument on invocation of the procedure, and
             when execution of the procedure completes, any pointers that do not become undened (16.5.2.5) and are associated with the dummy argument remain associated with the effective argument.

Section 12.5.2.7 Pointer dummy variables
The requirements in this subclause apply to actual arguments that correspond to dummy data pointers.
C1241 The actual argument corresponding to a dummy pointer with the CONTIGUOUS attribute shall be simply contiguous (6.5.4).

Section 12.5.2.8.2 Coarray dummy variables
If the dummy argument is an array coarray that has the CONTIGUOUS attribute or is not of assumed shape, the corresponding actual argument shall be simply contiguous.

The following constraints also be identified in standard. The diagnostic test cases will be created  based on them.

Constraints

- C448 (R436) If the CONTIGUOUS attribute is specied, the component shall be an array with the POINTER Attribute.
- C530 An entity with the CONTIGUOUS attribute shall be an array pointer or an assumed-shape array.

- C1239 (R1223) If an actual argument is a nonpointer array that has the ASYNCHRONOUS or VOLATILE attribute but is not simply contiguous (6.5.4), and the corresponding dummy argument has either the VOLATILE or ASYNCHRONOUS attribute, that dummy argument shall be an assumed-shape array that does not have the CONTIGUOUS attribute.

- C1240 (R1223) If an actual argument is an array pointer that has the ASYNCHRONOUS or VOLATILE attribute but does not have the CONTIGUOUS attribute, and the corresponding dummy argument has either the VOLATILE or ASYNCHRONOUS attribute, that dummy argument shall be an array pointer or an assumed-shape array that does not have the CONTIGUOUS attribute.

- C1241 The actual argument corresponding to a dummy pointer with the CONTIGUOUS attribute shall be simply contiguous (6.5.4).

Test Strategy

The IT1 will cover functional test and diagnostic test. The functional test will combine the following :

1. Apply CONTIGUOUS attribute to all intrinsic types

 

2. Apply CONTIGUOUS attribute to derived types with and without type parameters. This is to test specifically the compiler dependent contiguity. The XLF compiler considers contiguous objects containing a single component that have no rta (run-time padding). The following illustrates the different scenarios:

TYPE Exp1(n)
	integer, LEN :: n
	character(n) :: name
END TYPE
Type(Exp1(4)) :: dt4(10)
Type(Exp1(3)) :: dt3(10) 
 


dt3%name is not contiguous


dt4%name is contiguous 


 


This is a compiler dependent behavior and needs to be tested.


3. An array pointer or assumed-shape array with CONTIGUOUS attribute can be
- 1-dim, 2-dim, 3-dim and multi-dim array
- local variable  (for array pointer only)
- module variable (for array pointer only)
- derived-type component (for array pointer only)
- function result (for array pointer only)
- dummy arg
- actual arg

4.  An array pointer or assumed-shape array with CONITUGOUS can appear in
- an expr
- array constructor
- structure-constructor as data-target (for array pointer component only)

5. Passing contiguous arrays in procedure calls (FUNCTION/SUBROUTINE): 

  • Internal/External and Module procedures
  • Actual argument being simply contiguous and dummy is an assumed-shape array with CONTIGUOUS attribute
  • Actual argument being a pointer with CONTIGUOUS attribute and dummy argument  an assumed-shape array with CONTIGUOUS attribute
  • Dummy argument being an array pointer with CONTIGUOUS attribute is (argument) asscoiated with a non-pointer actual contiguous arg or a pointer that is asscociated with contiguous target
  • Assumed-shape array dummy argument with TARGET and CONTIGUOUS attributes and  the actual argument which is an array pointer with CONTIGUOUS attribute
  • Assumed-shape array with TARGET and CONTIGUOUS attributes and the actual argment is nonpointer contiguous array
  • Assumed-shape array with INTENT(IN) and CONTIGUOUS attributes and the actual argment is nonpointer contiguous array
  • Assumed-shape array coarray dummy argument that has the CONTIGUOUS attribute, the actual argument is contiguous array
  • Assumed-shape array with CONTIGUOUS attribute in a recursive Subroutine where the shape changes at each recursive call
  • Test a pointer with CONTIGUOUS attribute asscoiated with actual simply contiguous arg that has TARGET attribute becomes associated with dummy arg of assumed-shape array with CONTIGUOUS and TARTET attributes. It becomes associated with actual arg again after execution of procedure completes.
  • Deeply nested procedures  

 

6. In a data pointer assignment, an array pointer has CONTIGUOUS attribute and target is
- nonpointer nonassumed-shape whole array
- or simply contiguous array section (i.e.  A(:,3), A(:,d+1),A(:,lb:ub), A(:,:5))
- or a pointer and assumed-shape array with CONTIGUOUS attribute
- or a pointer (which has no CONTIGUOUS attr) asscoiated with contiguous target
- or assumed-shape array ( which has no CONTIGUOUS attr) asscoiated with contiguous actual
- or function result which is contiguous array pointer

7. Test bound-remapping, the target is array pointer or assumed-shape array with CONTIGUOUS attribute. i.e.
Ptr(1:size(tar)) => tar(lb1:,:ub2) where 2-dim tar has CONTIGUOUS attr
Ptr(I:J,K:L) => tar  where ptr has CONTIGUOUS attr and tar is 1-dim (I,J,K,L are INTEGER)

8. Test an array pointer with CONTIGUOUS attribute is associated with contiguous target by allocate statement; Test data pointer assignment appear inside nested  do-loop, where data-pointer (it may be DT component or object)  has CONTIGUOUS attr

9. Test selectors of ASSOCIATE and SELECT TYPE construct are array pointers or assumed-shape arrays with CONTIGUOUS attribute. Polymorphism will be tested only in this case


For intrinsic IS_CONTIGUOUS(ARRAY)
 

10. Test ARRAY is of
- intrinsic types
- derived-types (with and without DTP). The Focus will be on the compiler dependent contiguity.

- unpolymorphic DT
- polymorphic DT

11*. Test IS_CONTIGUOUS(ARRAY) has value .false. When ARRAY is the following
- pointer which is not associated
- or allocatable array which is not allocated
- or zero-sized array
- or function result which is non associated pointer array
- or array section i.e. B(:,3,:), C(::2)
- or arr%x where arr is array of derived-type and has x and y components
- or substring of scalar variable of type character
- or substring of array variable of type character
- or the associating entity of SELECT TYPE and ASSOCIATE constructs and the selector is not contiguous
- or dummy argument which actual argument is not contiguous
- or data-pointer-object in pointer assignment where data-target is not contiguous i.e. p=>t(::2)
- or arr%x where arr is array of deiver-type which has only x component when -qautodbl is on

12*. Test IS_CONTIGUOUS(ARRAY) has value .true. When ARRAY is the following
- dummy argument which actual argument is contiguous
- or function result which is contiguous array
- or function result which is associated pointer array
- or data-pointer-object in pointer assignment where data-target is contiguous
- or array constructor i.e. is_contiguous((/1,2,3/))
- or an array with only 1 element
- or arr%x where arr is array of deiver-type which has only x component when -qautodbl off
- or the associating entity of SELECT TYPE and ASSOCIATE constructs and the selector is contiguous

13. IS_CONTIGUOUS(ARRAY) used in expression

*Items 11 and 12 will be tested with ARRAY arguments having a rank >=2


The diagnostic test will contain the following :

1. Test constraint C448
- apply contiguous attr to derived-type component which are allocatable array, scalar pointer or explicit array

2. Test constraint C530
- apply CONTIGUOUS attr to objects which are dummy arg (i.e. arg(lb:ub), arg(:ub)), scalar, scalar pointer, explicit array, assumed-size array,  implied-shape array or allocatable array,
- apply CONTIGUOUS attr to function return which is allocatable array array, explicit array  or scalar pointer

3. Test constraint C1239
- test dummy arg is assumed-shape array with CONTIGUOUS and ASYNCHRONOUS attr and actual arg is nonpointer array that has ASYNCHRONOUS attr and is not simply contiguous
- test dummy arg is assumed-shape array with CONTIGUOUS and VOLATILE attr and actual arg is nonpointer array that has VOLATILE attr and is not simply contiguous

4. Test constraint C1240
- test dummy arg is assumed-shape array with CONTIGUOUS and VOLATILE attr and actual arg is pointer array that has VOLATILE attr and has no CONTIGUOUS attr
- test dummy arg is assumed-shape array with CONTIGUOUS and ASYNCHRONOUS attr and actual arg is pointer array that has ASYNCHRONOUS attr and has no CONTIGUOUS attr
- test dummy arg is array pointer with CONTIGUOUS and VOLATILE attr and actual arg is pointer array that has VOLATILE attr and has no CONTIGUOUS attr
- test dummy arg is array pointer with CONTIGUOUS and ASYNCHRONOUS attr and actual arg is pointer array that has ASYNCHRONOUS attr and has no CONTIGUOUS attr

5. Test constraint C1241
- test actual argument is not contiguous when dummy pointer has CONTIGUOUS attribute

6. For intrinsic IS_CONTIGUOUS(ARRAY), where ARRAY is
- scalar which is constant, variable, function result
- IS_CONTIGUOUS used in constant expr
