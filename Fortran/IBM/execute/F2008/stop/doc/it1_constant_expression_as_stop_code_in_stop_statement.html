Overview

%Note{Simple description of the feature; In which product it is first implemented, is it common for other platforms, is it common for both C and C++ and so on.}%

In Fortran 2008 the ERROR STOP and STOP statements [4] (Section 8.4) have been updated and now supports scalar-default-char-constant-expr and scalar-int-constant-expr as the stop code value. This document describes the design for testing these new additions to the ERROR STOP and STOP statements. The new message added for the EEROR STOP and STOP statements is as follows:

<filename>, line <21.11>: 1516-332 (S) The stop code in the <ERROR STOP / STOP> statement must be a scalar constant expression of character or integer type.
The parts in <> are variables.


Test Strategy

%Note{Test strategy used in the test}%

Following are some of the considerations while writing the new testcases (TCs):

  1. Different expressions generating a constant integer or character.
  2. Differences between ERROR STOP and STOP statements.
  3. Test in CAF and non-CAF mode.

Apart from the unit TCs available, provided by Tarique Islam, following new TCs will be added:


Diagnostics:

  • Testing for different constant integer and character expressions (LEN and KIND etc).
  • Testing for non constant integer and character expressions (loop counters etc).
  • Different variations of stop code values will be tested like: -10, +10, x+y*z, 10a and 6+10*(2*3) etc.
  • Testing the labeled stop statement.

Functional:

  • Testing if the program stops at the error stop or stop statement.
  • Testing the program exit status in CAF mode. The program exit status returned by the stop statement is (exit code: 0) different than the error stop statement (exit code: 1) as mentioned in [1] (Feature Overview --> Key Design Decisions).
  • Testing the Image number in CAF mode. When error stop or stop statement is used the runtime adds <Image image num> to the output when option errimagenum is set [1] (Feature Overview --> Key Design Decisions).
