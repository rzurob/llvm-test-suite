Feature Overview

The Fortran 2008 standard allows the :: separator in procedure and module procedure statements inside interface blocks. This is a change from Fortran 2003.

Key Requirements

Support new syntax and perform language level checking.

Key Design Issues

None.

Key Design Decisions

None.

Testing Strategy

This feature will be tested via unit tests only. Please refer to the testing considerations section for more information.

User Documentation

Overview

The Fortran 2008 standard allows the :: separator in procedure and module procedure statements inside interface blocks. This is a change from Fortran 2003.

User Benefits

This is syntactic sugar that makes the syntax of module procedure statement more consistent with the syntax of other declaration statements.

Public Interface

Procedure statement now has this syntax diagram:

>>-+--------+--PROCEDURE--+----+--procedure-name_list----------><
   '-MODULE-'             '-::-'(1)

(1): Fortran 2008.

Constraints

None.

Rules

None.

Language Standard Considerations

Allowing :: in the procedure statement described in the syntax diagram above is a Fortran 2008 language feature. It should be flagged with language level checking for the Fortran 90, 95, and 2003 standards for the module procedure statement, and for Fortran 2003 for the procedure statement. (Omitting MODULE was a Fortran 2003 feature.)

Messages

A new message was reviewed and approved by the XLF Messages committee: "1518-387 (L) The double colon separator is not permitted in the PROCEDURE statement by the %1$s standard."

Samples

 
module m
  interface foo
    procedure x
    procedure :: y          ! New way
    module procedure x2
    module procedure :: y2  ! New way
  end interface
contains
  subroutine x(a, b)
    integer :: a, b
  end subroutine
  subroutine y(c)
    real :: c
  end subroutine
  subroutine x2(d, e, f)
    character(10) d, e, f
  end subroutine
  subroutine y2(g)
    complex(16) g
  end subroutine
end module

Testing Considerations
  • Test that the compiler now allows :: after procedure or module procedure in interface blocks.
  • Test that a langlvl message is issued under -qlanglvl=95std for module procedure followed by :: in interface blocks.
  • Test that a langlvl message is issued under -qlanglvl=2003std for procedure followed by :: in interface blocks.
  • Test that no langlvl message is issued under -qlanglvl=2008std for module procedure or procedure followed by :: in interface blocks.
  • Test that other F2003 ways of having procedure followed by :: are not flagged with the langlvl message for this feature. (e.g. procedure outside an interface block.)
