Overview

%Note{A brief description of the feature, its application(s) and limitation(s).

Feature syntax may also be mentioned here. Since it is in the I0 document, it is optional.}%

F2008 introduced extensions to the VALUE attribute:
"The value attribute is permitted for an array, and for an object with a nonconstant length type parameter."

This IT1 is to test support for assigning the VALUE attribute to dummy arguments for
- Explicit shape arrays of various type, kind & rank
- Assumed shape arrays of various type, kind & rank

This IT1 will NOT be considering objects with a nonconstant length type parameter.

Functional Tests

The general test strategy for the feature will involve arrays being passed to procedures. Within the procedures, the dummy arguments for the arrays will have the VALUE attribute. Testing will check that the dummy arguments match the actual arguments, and that no changes done to the dummy arguments will affect the actual argument.

The actual arguments can be:
  •     array constructors
  •     array variable
  •     array section
  •     array constants
  •     both contiguous and noncontiguous arrays
The dummy argument can be:
  •    explicit shape array
  •    assumed shape array
Dummy arguments must also be considered using argument keywords

The scopes to consider are:
  •     external procedure
  •     module procedure
  •     internal procedure
Procedures for arrays can be:
  •     either a subroutine or function
  •     nested
  •     pure

Arrays can have the following attributes:

  • CONTIGUOUS
  • DIMENSION
  • OPTIONAL
Arrays can be declared using the dimension attribute or by explicitly specifying its rank and shape.
Testing will involve arrays of the following types and kinds:
   
Type Kind
integer 1,2,4(default),8
logical
 1,2,4(default),8
real 4(default),8,16
complex 4(default),8,16
character 1

Arrays of Derived Types:
  • components of intrinsic data types
  • components of other allowed derived types
  • ALLOCATABLE component
  • POINTER component
  • derived types including the sequence type
Diagnostic Tests:

The following tests are expected to have an error returned by the compiler.

1) The array definitions definitions listed are not allowed with a dummy array using the VALUE attribute:
  • Assumed size array
  • Assumed rank array
  • Deferred shape array (allocatable/pointer)
2). The attributes listed are not allowed on a dummy array using the VALUE attribute:
  • POINTER  
  • ALLOCATABLE
  • VOLATILE  
  • INTENT(INOUT)
  • INTENT(OUT)
3) An array dummy argument with the VALUE attribute is not interoperable. The following cases testing interopability with C should also return an error:
  • Dummy array with the VALUE of a BIND(C) routine
