Overview

Fortran 2008 standard specifies the following constraints for PURE procedures that apply to the VALUE attribute:

"C1276 The specification-part of a pure function subprogram shall specify that all its nonpointer dummy data objects have the INTENT (IN) or the VALUE attribute.

C1277 The specification-part of a pure subroutine subprogram shall specify the intents of all its nonpointer dummy data objects that do not have the VALUE attribute."

There is also this NOTE following the list of all constraints for PURE procedures:

"NOTE 12.49
The above constraints are designed to guarantee that a pure procedure is free from side effects (modifications of data visible outside the procedure),..."

Test Strategy

The feature includes functional and diagnostic tests:

Functional:

- test the following types of dummy arguments with value attributes:
    * all intrinsic data types (see *1 below)
    * derived types including extended, nested and parameterized
    * C/Fortran interoperable types (see *2 below)

- dummy arguments of a pure subroutine include one or more of the attributes: value, intent(in), intent(out), intent(inout)
- dummy arguments of a pure function include one or more of the attributes: value, intent(in)
- derived type dummy arguments include components that are explicit, as well as allocatable, arrays
- array actual argument passed to elemental function with scalar dummy arg having value attribute; include max rank (20) array actual argument
- pure procedures are also type-bound procedure components
- pure procedures include: pure functions, pure subroutines, elemental functions, elemental subroutines
- all pure procedures verify the values in all dummy arguments
- all pure procedures return/set a result, caller verifies the result
- all pure procedures modify all dummy arguments that are declared with value attribute; on return, caller verifies corresponding actual arguments are unchanged
- pure procedures defined in modules, internal and external

Diagnostic:

Verify the correct compiler error is emitted for these cases:
- use value attribute in dummy arguments of a defined operator
- declare dummy data objects with neither intent(in) nor value
- declare dummy procedure with value attribute, including nested functions and subroutines
- declare dummy data objects with value attribute and one of: POINTER, ALLOCATABLE, VOLATILE, INTENT(INOUT), or INTENT(OUT) attributes
- specify value attribute more than once for a dummy data object
- attempt to modify intent(in) dummy argument that also has value attribute
- declare dummy data objects as arrays with value attribute

*1: intrinsic data types tested:

- INTEGER (kind=default,1,2,4,8)

Test Procedures/Scenarios

Functional unit tests:

The following is common to the testcases below:
- pure procedures include: pure functions, pure subroutines, elemental functions, elemental subroutines
- pure procedures verify the values in all dummy arguments
- pure procedures return/set a result, caller verifies the result
- pure procedures modify all dummy arguments that are declared with value attribute; on return, caller verifies corresponding actual arguments are unchanged

purevalue01.scenario (from Rafik)
- pure procedures with integer dummy arguments having value attributes, and others with intent(out)
- this testcase also include a diagnostic step, in which the source file is compiled with -qlanglvl=2003std; error expected:
  1513-262 (L) Dummy arguments of pure procedures are not permitted to have the VALUE attribute by the Fortran 2003 standard.

purevalue02.scenario (from Rafik)
- dummy arguments are parameterized derived types that include type-bound procedure components
- test that we can pass a by-descriptor object by value to a pure subroutine

purevalue03.scenario
- several pure procedures with mixed # of dummy arguments with value attribute preceded and succeeded by intent(in|out|inout)

purevalue04.scenario
- pass all intrinsic data types as dummy args with value attributes to pure procedures
- pass higher than average # (32) of dummy arguments with value attribute to pure procedures

purevalue05.scenario
- same as purevalue04.f, except dummy arguments that are declared with value attribute are also declared as intent(in)
- same behavior is expected

purevalue06.scenario
- dummy data objects with value attribute before and after a procedure dummy argument
- the dummy procedure is also a pure function that takes a dummy argument with value attribute, also modifies its value dummy arguments

purevalue07.scenario
- dummy data objects with value attribute before and after extended derived type dummy argument
- the derived type has an explicit as well as an allocatable rank-20 array that are modified in the pure procedures

purevalue08.scenario
- rank-20 array actual argument passed to elemental function with scalar dummy arg having value attribute
- function modifies dummy arg and returns an array
- caller verifies actual argument array is not modified, and the array returned
- repeat test by passing pointer to the array to the same elemental function

purevalue09.scenario
- C main calls Fortran pure procedures, passing all interoperable intrinsic types, declared with value attribute

purevalue10.scenario
- C main calls Fortran pure function, passing a nested struct as argument, dummy argument derived type is defined with value attribute

purevalue11.scenario
- C main calls Fortran pure procedures, passing a int*
- dummy arg is type(c_ptr) with value attribute
- procedures attempts to modify target in C's pointer
- C main verifies its value remains unchanged (see interp defect 384751 for this testcase)

purevalue12.scenario
- C main calls Fortran pure procedures, passing pointer to a C function
- dummy arg is type(c_funptr) with value attribute

purevalue13.scenario
- pass vector data types as dummy arguments with value attributes to pure procedures

purevalue14.scenario
 - this is a copy of purevalue03.f, modified to define the pure procedures externally in a separate file, and declare them through explicit interface, module interface, and use association

purevalue15.scenario
- this is a copy of Rafik's purevalue02, extended to include pure functions as type-bound procedure components, with dummy arg having value attribute

Diagnostic tests

The following is common to the testcases below:
- pure procedures include: pure functions, pure subroutines, elemental functions, elemental subroutines

purevalue01d.scenario
- use value attribute in dummy arguments of a defined operator; error expected:
  "1514-711 (S) Dummy argument a of procedure or binding foo must be specified with the INTENT(IN) attribute. foo will not be included in the OPERATOR(.def_op.) generic interface."

purevalue02d.scenario
- declare dummy data objects with neither intent(in) nor the value attribute; error expected:
* for pure/elemental functions:
   "1513-164 (S) A dummy argument of a pure function that does not have the POINTER attribute and is not a dummy procedure must have the INTENT(IN) or VALUE attribute"
* for pure/elemental subroutines:
   "1513-165 (S) A dummy argument of a pure subroutine must be given the INTENT or VALUE attribute, unless it is a dummy procedure, it is an asterisk, or it has the POINTER attribute."

purevalue03d.scenario
- declare dummy procedure with value attribute; error expected:
  "1516-178 (S) Dummy procedure <x> must not be declared with the VALUE attribute."
- test above with also nested functions and subroutines

purevalue04d.scenario
- declare dummy data objects with value attribute and one of: POINTER, ALLOCATABLE, VOLATILE, INTENT(INOUT), or INTENT(OUT) attributes; error expected:
  "1516-174 (S) Objects with the VALUE attribute must not be declared with the POINTER, ALLOCATABLE, VOLATILE, INTENT(INOUT), or INTENT(OUT) attributes."
- specify value attribute more than once for an object; error expected:
  "1514-180 (W) An entity must not be given explicitly any attribute more than once in a scoping unit"
- attempt to modify intent(in) dummy argument that also has value attribute; error expected:
  "1516-055 (S) The INTENT(IN) attribute specifies that a dummy argument, or a subobject of a dummy argument, must not be redefined or become undefined during the execution of the procedure."

purevalue05d.scenario
- declare dummy data objects as arrays with value attribute; error expected:
  "1516-177 (S) Arrays must not be declared with the VALUE attribute."

purevalue06d.scenario
- same as purevalue02d.scenario, except this uses vector type
