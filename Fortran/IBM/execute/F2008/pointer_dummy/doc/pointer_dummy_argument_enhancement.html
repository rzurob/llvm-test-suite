Feature Overview

Key Requirements

Fortran 2008 allows procedure calls where:
1. The data dummy argument has the POINTER and INTENT(IN) attributes, and the corresponding actual argument is a
non-pointer that has the TARGET attribute.

2. The dummy argument is a procedure pointer has the INTENT(IN) attribute, and the corresponding actual argument
is a procedure pointer, a reference to a function that returns a procedure pointer, a reference to the intrinsic function NULL, or
a valid target for the dummy pointer in a pointer assignment statement.

It is complied with the Fortran 2008 standard.

Key Design Issues

(1) What code should be generated by the compiler when the dummy argument has the POINTER and INTENT(IN) attributes
and the actual argument is a non-pointer that has the target attribute?

(2) What code should be generated by the compiler when the dummy argument is a procedure pointer with INTENT(IN) and the
actual argument is a non-pointer that is a valid target for the dummy pointer in a pointer assignment statement?

Key Design Decisions

(1)The compiler is supposed to create a temporary pointer, pointer assign it to the actual argument, and pass it to the procedure instead of the actual argument.

(2)Similar to the implementation for data pointer, the compiler is supposed to create a temporary procedure pointer, the actual argument is pointer associated with the
newly created temp pointer, and then it is passed to the procedure instead of the actual argument.

Testing Strategy

This feature is only tested via unit testing:
- Functional:
(1)Test that when the dummy argument has the POINTER and INTENT(IN) attributes of a procedure, and the actual argument is a non-pointer that has the
TARGET attribute, we can get the expected result if calling this procedure.

- Diagnostic:
(1)Test that when the condition of pointer dummy argument enhancement is not satisfied, expected error message should be issued(1513-061).
(2)Test that when the condition of pointer dummy argument enhancement is satisfied, and -qlanglvl=<older standards> is specified, a langlvl message(L) is issued.
User Documentation

Overview

Fortran 2008 allows procedure calls where:
1. The dummy argument has the POINTER and INTENT(IN) attributes, and
2. The actual argument is a non-pointer that has the TARGET attribute

User Benefits

Users now can pass a non-pointer actual argument that has the TARGET attribute to a procedure in which, the dummy argument has the
POINTER and INTENT(IN) attributes.

Public Interface

None.

Constraints

(1)If a data dummy argument has the POINTER and INTENT(IN) attributes, the corresponding actual argument must have
either the POINTER or the TARGET attribute.
(2)If the dummy argument is a procedure pointer, and the corresponding actual argument is not a pointer, the dummy argument shall
have the INTENT(IN) attribute.

Rules

None.

Language Standard Considerations

When the condition of pointer dummy argument enhancement is satisfied, and -qlanglvl=<older standards>(before F2008) is specified,

a langlvl message(L) is issued.

Messages

When the condition of pointer dummy argument enhancement is satisfied, and -qlanglvl=<older standards>(before F2008) is specified,
we issue the following langlvl message:

"(L) Dummy argument %1$s of the procedure %2$s has the %3$s attribute. The corresponding actual argument is required to also have the
%3$s attribute by the %4$s standard."

Samples

Data pointer:

program t
interface
subroutine sub(p)
integer, pointer, intent(in) :: p ! The dummy argument has the POINTER and INTENT(IN) attributes
end subroutine
end interface

integer, target :: t ! The actual argument is a non-pointer that has the TARGET attribute
t = 5
call sub(t) ! Should print p is: 5

end

subroutine sub(p)
integer, pointer, intent(in) :: p

print *, "p is: ", p ! p is 5
end subroutine
Procedure Pointer:

 program p
   interface
     subroutine print_hello(c)
       character(*) :: c 
     end subroutine
   end interface
 
   call sub(print_hello) ! Should print "hello world"
 
   contains
     subroutine sub(dmy_pp)
       procedure(print_hello), pointer, intent(in) :: dmy_pp
       character(15) :: c
 
       c = "hello world"
 
       call dmy_pp(c)
     end
 end
 
 subroutine print_hello(c)
   character(*) :: c
 
   print *, c 
 end

Testing Considerations

General Testing will be focusing on the following areas:
- Pass the non-pointer actual argument that has the TARGET attribute to a procedure in which
the dummy argument has the POINTER and INTENT(IN) attributes.
- Procedure pointer as actual and dummy argument.
- Scalar, array, derived type, characters pointer as actual and dummy argument.
- Langlvl testing.
- Diagnositc testing with different error condition.

High-Level Implementation Design

Implementation Design Overview

(1)Remove the check constraint: Non-pointer actual argument that has the TARGET attribute doesn't match the dummy
argument has the POINTER and INTENT(IN) attributes.

(2)Create the temporary pointer that points to the non-pointer actual argument that has the TARGET attribute. i.e.

interface
subroutine foo(p)
integer, pointer, intent(in) :: p 
     end subroutine 
end interface
integer, target :: t
t = 5
call sub(t) 
 ! integer, pointer :: temp_p
 ! temp_p => t
 ! call sub(temp_p) 
 end

(3)For procedure pointer, we create a temporary procedure pointer, the actual argument then is pointer associated with the
newly created temp pointer, and then it is passed to the procedure instead of the actual argument.

 program p
   interface
     subroutine print_hello(c)
       character(*) :: c 
     end subroutine
   end interface
 
   call sub(print_hello) ! Should print "hello world"
 
   ! procedure(print_hello), pointer :: temp_p 
   ! temp_p => print_hello 
   ! call sub(temp_p) 
   contains
     subroutine sub(dmy_pp)
       procedure(print_hello), pointer, intent(in) :: dmy_pp 
       character(15) :: c 
       c = "hello world"   
       call dmy_pp(c)
     end
 end
 
 subroutine print_hello(c)
   character(*) :: c
 
   print *, c
 end
