%Note{IT2 location and other related information}%

Permanent location of the TCs:

   F2008/complexPartDesignators/


The following tables list the details of all the TCs:


Diagnostic:

Only one of the error/warning messages is listed as example. Detail list of messages can be checked in the verification files.

Scenario Description Results
complexPartDesignatorD1.scenario Test with different syntaxes.
 Verify the error messages in the output file:
1515-038 (S) Only an object with the POINTER attribute is permitted on the left hand side of a pointer assignment.
complexPartDesignatorD2.scenario Test with different data types, logical, character, integer, real and complex. Also test the type of the complex-part-designator in an argument.
 Verify the error messages in the output file:
1516-343 (S) A complex part designator must only appear on a complex type.
complexPartDesignatorD3.scenario Test the kind of complex-part-designator.
 Verify the results in the TC with known values.
complexPartDesignatorD4.scenario Test the use in allocate and pointer statements both as scalar and array. Test the use in equivalence statement as equivalence object. Verify the error messages in the output file:
1515-019 (S) Syntax is incorrect.
1514-140 (E) EQUIVALENCE statement ignored due to presence of syntax errors.
mplexPartDesignatorD5.scenario Check different levels (77, 90, 95, 2003 and 2008) of the Fortran language. Verify the error messages in the output file. Any level below Fortran 2008 should return a warning message:
1518-376 (L) Complex part designators are not permitted by the Fortran <level> standard.

Functional:

Scenario Description Results
complexPartDesignatorF1.scenario
 Test the use as an actual argument, as scalar, in a module, a function and a subroutine. Verify the results with known values and using the intrinsics real() and imag() in the TC. Before moving to testing complex part designators it test the real() and imag() intrinsics.
complexPartDesignatorF2.scenario
 		Test the use as an actual argument, as array, in a module, a function and a subroutine. Verify the results using real() and imag() intrinsic functions in the TC. Before moving to testing complex part designators it test the real() and imag() intrinsics.
complexPartDesignatorF3.scenario
 		Test the use in different expressions, initialization, constant and assignment including double precision numbers. Verify the results using real() and imag() intrinsic functions, and known values in the TC.
complexPartDesignatorF4.scenario
 		Test the use in allocate and pointer assignment statements both scalar and array. Test the target as a complex-part-designator in pointer statement both scalar and array.
 Verify the results with known values in the TC.
complexPartDesignatorF5.scenario
 		Test if an array section as a complex part designator is contiguous or not. Test using different sections of the array.
 Verify the results with the intrinsic function is_contiguous() in the TC.
complexPartDesignatorF6.scenario
 		Test the use in assignment statement as variables. Initialize real arrays (of size 4, 8 and 16) and assign them to real and imaginary parts of complex arrays using the complex part designators.
 Verify the results using Trignometric functions sin, cos and tan in the TC. Validation of the results is done using precision functions.
complexPartDesignatorF7.scenario
 		Test the use as an actual argument in an elemental function. Verify the results using real() and imag() intrinsic functions in the TC.
complexPartDesignatorF8.scenario
 Test the use in structure component, array element and array sections. Verify the results with known values in the TC and the verification file.
complexPartDesignatorF9.scenario
 Test the use in derived types. Verify the results using the verification file.
complexPartDesignatorCoarray1.scenario
 Test the use in array co-array with use of co-rank.
 Verify the results with known values in the TC.
complexPartDesignatorCoarray2.scenario
 Test the use in array co-indexed objects with use of co-rank. Verify the results with known values in the TC.
complexPartDesignatorCoarray3.scenario
 Test the use in scalar co-array and co-indexed objects. Verify the results with known values in the TC.
