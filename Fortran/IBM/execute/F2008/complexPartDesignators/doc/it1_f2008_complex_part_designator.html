Overview

%Note{Simple description of the feature; In which product it is first implemented, is it common for other platforms, is it common for both C and C++ and so on.}%

Fortran 2008 standard added a designator as complex part designator. This designator allows direct access to the real and the imaginary part of a complex number as defined in Section 6.4.4 of the Fortran 2008 standard [1]. This document describes the testing of this complex part designator.

Test Strategy

%Note{Test strategy used in the test}%

Following are some of the considerations while writing the new testcases (TCs). Example code is given as much as possible. Variables defined once are used throughout the example code. The example code given does not cover all the tests. For code that covers all the tests please see the IT2 [2]:

Diagnostic:

  • Test with different syntaxes:
     real r
     complex(8) c1, c2
     r = (1.0,0.0)%RE                     ! Syntax error
     c1%RE => 1.5                         ! Syntax error
  • Test with different types of designators, like integer, real, character and logical etc:
     r%RE                                 ! Type error
     r%IM                                 ! Type error
  • Test type and kind of the complex part designator:
     print *, KIND(c1%RE)
     print *, KIND(c2%IM)                 ! Should print 8 in both cases
  • Test and check all the warning messages as approved by the XLF messages committee including the warning message when any language level lower than F2008 is used.
  • Test the use in allocate and pointer assignment statements:
     complex, allocatable :: a
     complex, pointer :: p
     complex, target :: t
     allocate(a%RE)                       ! Invalid
     allocate(a%IM)                       ! Invalid
     p%RE => t                            ! Invalid
     p%IM => t                            ! Invalid
  • Test the use in equivalence statement as equivalence objects:
     EQUIVALENCE (r, c1%RE), (r, c1%IM)   ! Invalid

Functional:
  • Test the use in CAF environment including co-arrays and co-indexed objects with different ranks. The example below is for scalar co-arrays:
     real, save :: R[*]
     complex, save :: C[*]
     if (this_image() == 1) then
        C[1]%RE = 1.3
        C[1]%IM = 1.2
     end if
     print *, C[1]                        ! Should print (1.3,1.2)
     SYNC IMAGES(1)                       ! Wait for image 1 to finish
     R = C[1]%RE
     print *, R                           ! Should print 1.3 for all images
     R = C[1]%IM
     print *, R                           ! Should print 1.2 for all images
  • Test the use in structure component, array element and array sections.
  • Set and test the real and imaginary parts:
     c1%RE = 1.5
     c1%IM = 0.0
     c2%RE = c1%IM
     c2%IM = c1%RE
     print *, c1                          ! Should print (1.5,0.0)
     print *, c2                          ! Should print (0.0,1.5)
  • Test the use in different expressions, initialization, constant and assignment etc:
     complex, parameter :: complex_value = (0.75,0.5)
     complex complex_number
     complex_number%RE = complex_value%RE ! assignment
     complex_number%IM = complex_value%IM ! assignment
     complex_number == complex_value
  • Test the use in allocate and pointer assignment statements. Also test the use of complex part designators as the target of a pointer assignment statement:
     allocate(a)
     a%RE = 1.5
     a%IM = 0.5
     p => t
     p%RE = a%RE
     p%IM = a%IM
     print *, p                           ! Should print (1.5,0.5)

     real, pointer :: p_r
     t = (3.3, 1.3)
     p_r => t%RE
     print *, p_r                         ! Should print 3.3
     p_r => t%IM
     print *, p_r                         ! Should print 1.3
  • Check if an array section as a complex part designator is contiguous or not:
     complex, dimension(7) :: c
     is_contiguous(c%RE)                  ! Should return false
  • Consider the following when verifying the designated value (single and double precision):
  • Compare the results in different expressions
       sin(c1%RE) == sin(1.5)             ! Should be exactly the same
  • Test the value designated using the intrinsic functions REAL and IMAG etc:
       c%RE == real(c)
       c%IM == imag(c)
  • Compare against the already known value
