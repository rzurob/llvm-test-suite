IT2 Comments

"ofsk" stands for Omitting Function and Subroutine Keywords


Function Testing:

Scenario Name Description
ofsk01f.scenario A simple program with only one single internal procedure whose Function is terminated by "END"
ofsk02f.scenario A simple program with only one single internal procedure whose Subroutine is terminated by "END"
ofsk03f.scenario A simple program with a minimal module whose Function is terminated by "END"
ofsk04f.scenario
 A simple program with a minimal module whose Subroutine is terminated by "END"
ofsk05f.scenario A subroutine and a pure function within a module are terminated by "END"
ofsk06f.scenario Two internal subroutines both are terminated by "END"
ofsk07f.scenario Eleven internal functions are all terminated by "END"
ofsk08f.scenario Five out of Eleven internal functions are terminated by "END", and the rest is terminated by "END FUNCTION"
ofsk09f.scenario For internal procedures, five subroutines and one functions are terminated by "END"
ofsk10f.scenario For internal procedures, three subroutines are terminated by "END SUBROUTINE", and two other subroutine and one function are terminated by "END"
ofsk11f.scenario Internal recursive subroutine is terminated by "END"
ofsk12f.scenario Two modules, all the module functions and subroutines in them are terminated by "END"
ofsk13f.scenario Two modules, some module functions and subroutines are terminated by "END", the rest are terminated by "END SUBROUTINE" or "END FUNCTION"
ofsk14f.scenario Nested internal function within module's function is terminated by "END", and the module procedure (function) is terminated by "END" too.
ofsk15f.scenario Nested internal subroutine within module's subroutine is terminated by "END", and the module procedure (subroutine) is terminated by "END" too.
ofsk16f.scenario Nested internal subroutine within module's function is terminated by "END", and the module procedure (function) is terminated by "END"
ofsk17f.scenario Nested internal function within module's subroutine is terminated by "END", and the module procedure (subroutine) is terminated by "END SUBROUTINE"

Diagnosis Testing

Scenario Name Description
ofsk01d.scenario Error message expected when using "END blah" to terminate an internal function called "blah"
ofsk02d.scenario Error message expected when using "END blah" to terminate an module subroutine called "sub"
ofsk03d.scenario Error message expected when it has three-level nested internal procedures
