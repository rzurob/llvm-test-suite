Overview

 

 
  Fortran 2008 allows omission of the FUNCTION or SUBROUTINE keyword in the END statement of an internal procedure or a module procedure.
      E.g. end-function-stmt is END [ FUNCTION [ function-name ]]

Test Strategy

The testcases that we will be using will simply be the ones that already exist in our test run scenarios, except taking out the "FUNCTION" or "SUBROUTINE" keyword followed by "END" in the internal or module procedures.


For the function verification testing, we will try to test different varieties/combinations of internal or module procedures to see if those testcases could be successfully terminated (no error output expected from compiler) by specifying "END" keyword.


For the diagnose verification testing, we will try to verify that the internal procedures or module procedures can not be terminated by "END function-name" when the "FUNCTION" OR "SUBROUTINE" keywords are not present.

 Since the feature only involves a syntax change, it won't have any appreciable impact on performance. Hence high optimization option testing can be safely omitted.
